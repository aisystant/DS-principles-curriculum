# Заготовка: Руководство 3 «IWE: работа и развитие»

> **Объект:** Машина (IWE — Репозитории, Среда, Агенты).
> **Ось:** Мастерство (M2 — управление IWE, M5 — создание IWE).
> **Актуально со ступеней:** 2 (работа), 3–5 (развитие).
> **Время на подраздел:** 2 часа (7 подразделов = 14 часов/неделю).
> **Не требует приложения «Через IWE»** — само является руководством про IWE.

---

## Основные идеи (из Pack)

1. **IWE ≠ инструмент. IWE = среда**, в которой работает Пилот. Руководство 3 — про среду, руководство 2 — про действия в среде.
2. **Болид-онтология:** Пилот + Машина = Созидатель. Без Machine Pilot не может реализовать свой потенциал.
3. **3 компонента Машины:** Репозитории (Pack, DS), Среда (VS Code, Git, хуки), Агенты (модели + MCP).
4. **ОРЗ** — явное начало и конец задачи: Открытие → Работа → Закрытие (PD.METHOD.010).
5. **Pack = SoT** — структура: FORM, METHOD, CAT, SOTA, PRINC. Drift-detection при изменениях.
6. **Агенты** — Навигатор, Портной, Диагност, Оркестратор. Не автопилот (D.046).
7. **Обратная связь от машины** — Аттестатор (bh) + Диагност (cp). Машина говорит правду, которую Пилот не хочет слышать.
8. **Развитие IWE** — Pack-дизайн, протоколы, MCP, расширение среды. Это M5 (мастерство создания IWE).

## Основные понятия (из Pack + текущих docs)

### Из Pack (обязательно в руководстве)
- IWE (интеллектуальное рабочее окружение)
- ОРЗ (Открытие — Работа — Закрытие)
- Pack / DS (личные репозитории)
- Агент (модель + настройки + инструменты)
- MCP (Model Context Protocol)
- Протокол / хук
- Week Close / Month Close
- Drift-detection
- cp.iwe, cp.cre, cp.auto, cp.exo, bh.inv, bh.met

### Из текущих docs (проверка — не забыть)
- **Экзокортекс / second brain** — все технологии, усиливающие мозг
- **Личная цифровая трансформация** — освоение экзокортекса и ИИ-инструментов
- **Киберличность** — расширенное восприятие личности через цифровые технологии
- **Ритуал входа в роль** — определение роли, метода, рабочего продукта перед работой
- **Стоп-момент** — точка осознанного выбора (FORM.051)
- **Full Kitting** — подготовка всех ресурсов перед началом
- **Drift / рассинхронизация** — когда docs отстают от Pack
- **Git / хуки / автоматизация** — инфраструктура IWE
- **Алерты / обратная связь** — когда машина сигнализирует о проблеме

### Риски дублирования
- **Экзокортекс** — здесь детально, в руководстве 1 — как контекст, в руководстве 2 — как инструмент.
- **ОРЗ** — здесь как система, в руководстве 2 — как применение к методам.
- **Ритуал входа в роль** — здесь как протокол, в руководстве 2 — как практика перед методом.

## Чего НЕ должно быть в руководстве 3 (границы)

- Нет методов M001–M008 (это руководство 2)
- Нет мировоззренческой дуги (это руководство 1)
- Нет системного разбора (это руководство 4)
- Нет курсовых элементов

# Замечания к разделу 06: Ступени и ворота развития

**Дата проверки:** 2026-05-17  
**Файл:** docs/ru/personal-new/1-1-systemic-self-development/s6-stages-and-gates/  
**Состав раздела:** 7 подразделов (6.01–6.07), ~280 мин суммарное время

---

## 1. Структура

**Сильные стороны:**
- Каждый подраздел придерживается единого шаблона: заголовок → время → анонс → одно предложение → сигнатура понятий → мем → определение → развитие мысли → метод → пример → типичная ошибка → степени мастерства → проверка себя → на практике → см. также → что дальше. Шаблон соблюдается строго, это создаёт предсказуемость и облегчает навигацию.
- Логика раздела выстроена последовательно: ступени → ворота → окна → узкое место → выбор ролей → квадрант компетенций → план перехода. Каждый подраздел опирается на предыдущие.

**Замечания:**
- **Нет вводного файла раздела.** Другие разделы (напр., s1) начинаются с файла, обозначающего контекст и мотивацию раздела. В s6 читатель сразу попадает в §6.01 без предисловия.
- **Нет итоговых файлов.** В разделах s1–s3 присутствуют `*.08-concepts.md`, `*.09-exercises.md`, `*.10-review-questions.md`, `*.11-section-conclusions.md`. В s6 они отсутствуют. Раздел обрывается на §6.07 без синтеза, контрольных вопросов и глоссария.
- **§6.05 тематически выбивается.** Раздел называется «Ступени и ворота», но §6.05 (Профориентация и выбор ролей) посвящён выбору ролей через неудовлетворенности — теме, которая логичнее принадлежит разделу 7 (ценности, намерения, неудовлетворенности). Внутри раздела 6 это выглядит как вставной фрагмент.

---

## 2. Понятность

**Сильные стороны:**
- Раздел «Мем, который снимается» эффективно обезоруживает предубеждения до введения понятия.
- Примеры из жизни конкретны, узнаваемы и достаточно детализированы (специалист с абонементом, психолог без формализованного метода, разработчик с «мышлением письмом»).
- Формат «Степени мастерства» (объясняю → умею → навык → мастерство) даёт читателю точку отсчёта для самооценки.

**Замечания:**
- **Термин «IWE» вводится без объяснения.** В §6.01 написано: «включая IWE и сообщество» — без расшифровки аббревиатуры. Для читателя, незнакомого с курсом, это непрозрачная отсылка. Если IWE раскрывается в более позднем разделе, здесь нужна хотя бы краткая глосса или сноска.
  > «Проактивный — это режим, в котором человек не просто развивается, а проектирует своё развитие как систему, включая IWE и сообщество.» (§6.01)

- **Смешение русского и английского термина для одного понятия.** В §6.04 концепт называется «Узкое место», но уже в §6.06 и §6.07 он обозначается английским «bottleneck» в теле текста, а не только в YAML-метаданных. Это создаёт терминологическую непоследовательность.
  > «Возьмите ваш bottleneck-метод из §6.04» (§6.06)  
  > «Bottleneck — одно узкое место» (§6.07)

- **Неопределённые внешние источники.** Многократные ссылки на «PD.FORM.089», «PD.FORM.022», «PD.METHOD.008» делают утверждения неверифицируемыми для читателя — документы недоступны публично. Если это внутренние документы системы, стоит давать краткое резюме цитируемого положения прямо в тексте, а не только реквизиты.

## 3. Полнота

**Замечания:**
- **Отсутствует раздел итоговых понятий.** Раздел вводит: «Ступень развития», «Мастерство», «Ворота развития», «Двойная проверка», «Окно ступени», «Застревание», «Узкое место», «Теория ограничений», «Профориентация», «Выбор ролей», «Связь ступеней и квадранта», «План перехода». Без сводного глоссария читатель не может проверить, все ли понятия усвоены.

- **Отсутствует блок контрольных вопросов по разделу.** Каждый подраздел содержит «Проверку себя», но итоговых вопросов по разделу 6 в целом нет.

- **Нет примеров застревания с конкретным вмешательством.** §6.03 описывает «застревание на ступени» и говорит, что оно «требует вмешательства», но не объясняет, что это за вмешательство конкретно. §6.04 частично отвечает через «узкое место», но связь между застреванием и диагностикой узкого места не замыкается явно.

- **§6.05 не связан с планом перехода.** §6.07 строит план перехода на основе элементов §6.01–6.04 и §6.06, но §6.05 (профориентация) в план перехода не интегрируется. Читатель не понимает, как выбор ролей влияет на план перехода между ступенями.

- **Нет раздела «Выводы по разделу».** Последний подраздел §6.07 заканчивается переходом к разделу 7, но не подводит итог раздела 6.

## 4. Соответствие методологии

**Сильные стороны:**
- Четыре признака ступени (слот, методы, продукты, обратная связь) используются последовательно и стабильно от §6.01 до §6.07 — хорошая концептуальная связность.
- Двойная проверка (поведение + понимание) точно соответствует принципу «мышление письмом» и анти-иллюзии компетентности из других разделов курса.
- Применение Теории ограничений (Голдратт) к личному развитию в §6.04 методологически корректно и оригинально.
- Связь квадранта компетенций со ступенями в §6.06 — сильный интегративный ход.

**Замечания:**
- **Числовые пороги для переходов не обоснованы.** §6.06 утверждает: «для перехода на Пробующий — минимум один метод на стадии Осознанная Компетентность; для Систематического — три метода». Эти пороги приводятся как факт, без объяснения, откуда они взяты и на каком основании. Это снижает доверие к конструкции.
  > «Для перехода на ступень "Пробующий" нужен минимум один метод на стадии "Осознанная Компетентность". Для перехода на "Систематический" — минимум три метода на этой стадии и один на "Неосознанная Компетентность".» (§6.06)

- **§6.05 ссылается на §7.02 (Неудовлетворенности), которые ещё не введены.** По логике повествования, читатель знакомится с неудовлетворенностями только в разделе 7. В §6.05 они используются как рабочий инструмент без предварительного введения:
  > «Берём неудовлетворенности, переводим их в роли, которые эти неудовлетворенности закрывают» (§6.05)  
  Этот подраздел либо должен быть перенесён после §7.02, либо должен давать краткое вводное определение «неудовлетворенности» прямо в тексте.

## 5. Практическая применимость

**Сильные стороны:**
- Каждый подраздел заканчивается разделом «На практике» с конкретным действием, которое читатель может выполнить немедленно.
- Шаблон плана перехода в §6.07 — лучший практический элемент раздела: конкретный, наблюдаемый, ограниченный по времени.
- Диагностика по 4 признакам (§6.01) — операциональна и применима без дополнительных инструментов.

**Замечания:**
- **Шаблон плана перехода не предоставлен в виде отдельного артефакта.** §6.07 описывает шаблон, но не даёт его в виде таблицы, которую читатель мог бы скопировать. Читатель должен самостоятельно воссоздать структуру из текстового описания — это лишний барьер.

- **Временные оценки подразделов одинаковы и ненадёжны.** Все 7 подразделов имеют одинаковое время: «25 мин чтение + 15 мин практика». Реальная трудоёмкость практических заданий существенно различается (диагностика по 4 признакам vs. полный план перехода). Единое время создаёт ложное ожидание.

- **Практики не связаны между собой нарастающим образом.** Читатель в §6.01 диагностирует ступень, в §6.02 составляет два списка, в §6.03 считает недели, в §6.04 строит таблицу 4×4, в §6.05 выписывает неудовлетворенности, в §6.06 составляет список методов, в §6.07 составляет план перехода. Каждый раз с чистого листа. Если бы практики были спроектированы как единое нарастающее задание (один рабочий документ, который пополняется), результат §6.07 был бы органичным итогом работы через весь раздел, а не ещё одним отдельным упражнением.

## 6. Конкретные замечания

### З-01. Аббревиатура IWE без расшифровки (§6.01)
> «проектирует своё развитие как систему, включая IWE и сообщество»

**Проблема:** Читатель встречает аббревиатуру без объяснения. Если IWE — центральный концепт курса, его нужно хотя бы упомянуть в скобках при первом появлении.  
**Предложение:** Добавить: «включая IWE (личную рабочую среду) и сообщество» с отсылкой на раздел, где IWE раскрывается подробно.

---

### З-02. Терминологическая непоследовательность: «узкое место» vs. «bottleneck» (§6.04, §6.06, §6.07)
> «Возьмите ваш bottleneck-метод из §6.04» (§6.06)

**Проблема:** §6.04 вводит русский термин «узкое место», но §6.06 и §6.07 переключаются на английский «bottleneck» в теле русскоязычного текста без объяснения синонимии.  
**Предложение:** Либо унифицировать до «узкое место» везде в тексте (а «bottleneck» оставить только в YAML-метаданных), либо при первом употреблении английского варианта явно указать: «bottleneck (узкое место)».

---

### З-03. Форвардная ссылка на §7.02 в §6.05
> «выдвигаем гипотезу о роли, проверяем её через минимальный эксперимент... берём неудовлетворенности, переводим их в роли» (§6.05)

**Проблема:** «Неудовлетворенности» — концепт раздела 7, который читатель ещё не изучил. Использование его как инструмента в §6.05 нарушает дидактическую последовательность.  
**Предложение (вариант 1):** Перенести §6.05 в раздел 7, после §7.02.  
**Предложение (вариант 2):** Добавить в §6.05 краткое рабочее определение: «Неудовлетворенность — это состояние, когда текущая реальность расходится с желаемой. Подробнее — в §7.02.»

---

### З-04. Числовые пороги переходов без обоснования (§6.06)
> «для перехода на "Пробующий" нужен минимум один метод на стадии "Осознанная Компетентность"; для "Систематический" — минимум три метода»

**Проблема:** Числа 1 и 3 предъявлены как факт. Читатель не понимает, откуда они берутся и почему именно такое количество.  
**Предложение:** Добавить хотя бы одно-два предложения с логикой: «Три метода — минимум, обеспечивающий устойчивость системы: если один метод временно выпадает (болезнь, командировка), система не разрушается». Или честно указать: «Эти пороги — рабочая гипотеза программы, а не эмпирически выверенный стандарт».

---

### З-05. Отсутствует шаблон плана перехода в виде таблицы (§6.07)
> «Заполните шаблон плана перехода: (1) Цель-ступень... (2) Текущая ступень... (3) Bottleneck... (4) Действия... (5) Метрики... (6) Дата проверки» (§6.07)

**Проблема:** Шаблон описан списком в тексте, но не представлен визуально. Читатель должен мысленно перевести текст в таблицу.  
**Предложение:** Добавить после описания готовую таблицу-шаблон:

| Компонент | Заполнить |
|---|---|
| Цель-ступень | |
| Текущая ступень | |
| Узкое место (bottleneck) | |
| Действия (3–5) | |
| Метрики | |
| Дата проверки | |

---

### З-06. Практики не образуют единый нарастающий рабочий документ
**Проблема:** Читатель 7 раз создаёт новый документ/таблицу вместо того, чтобы поэтапно заполнять один. Это создаёт фрагментацию вместо интеграции.  
**Предложение:** Ввести в §6.01 единый «Рабочий лист раздела 6», в котором предусмотрены поля для всех практик, и указывать в каждом §6.0X: «Заполните строку X в рабочем листе».

---

### З-07. Раздел не имеет структурных итоговых файлов
**Проблема:** Разделы s1–s3 завершаются файлами `concepts.md`, `exercises.md`, `review-questions.md`, `section-conclusions.md`. В s6 они отсутствуют. Читатель лишён возможности сводной самопроверки.  
**Предложение:** Добавить минимум два файла:
- `6.08-concepts.md` — глоссарий раздела 6 (8–10 понятий)
- `6.09-section-conclusions.md` — итоги раздела и переход к разделу 7

## Итог

**Сильные стороны раздела:**
1. Строгая концептуальная логика: каждый подраздел работает на общую цель — научить читателя диагностировать и планировать переход между ступенями.
2. Последовательное применение четырёх признаков ступени (слот, методы, продукты, обратная связь) создаёт устойчивый каркас.
3. Практические задания в каждом подразделе конкретны и выполнимы.
4. Формат «Мем, который снимается» методологически точно работает с блокирующими убеждениями.

**Приоритетные замечания (в порядке важности):**
1. **З-07** — Добавить итоговые файлы раздела (concepts + section-conclusions). Критично для учебной целостности.
2. **З-03** — Устранить форвардную ссылку на §7.02 в §6.05. Нарушает дидактическую последовательность.
3. **З-06** — Спроектировать единый нарастающий рабочий документ для практик раздела.
4. **З-05** — Добавить визуальный шаблон плана перехода в §6.07. Низкая стоимость, высокий эффект.
5. **З-02** — Устранить смешение «узкое место» и «bottleneck» в теле текста.

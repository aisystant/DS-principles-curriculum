## Детализация

### 🔴 04-structure-guide-4.md — формат «Понятия:»

**Проблема.** Текущий формат каждой записи:
```
**Понятия:**
- Различение (операция, фиксирующая границу между понятиями) - Различение -> основа мышления -> единица анализа
- Граница понятий (линия раздела, не данная природой) - Граница -> различение -> фокус
```

Парсер STRUCTURE-ALL-GUIDES.md обрезает по `(` и теряет имя понятия — в STRUCTURE-ALL-GUIDES получается:
```
- Понятия: фиксирующая границу между понятиями); не данная природой); прошивающих интеллект)
```

**Лечение.**
1. Стандартизовать формат: имя — главное, скобки — второстепенное:
   ```
   **Понятия:**
   - **вводится:** Различение → `U.Episteme` (FPF A.7) — операция, фиксирующая границу
   - **используется:** Граница (см. 4.S3.SS2) — линия раздела
   - **противопоставлен:** Интеллект-стек — структура трансдисциплин
   ```
2. Удалить format-смешение «короткое определение в скобках» + «связи через стрелки».
3. Применить ко всем 4 structure-guide'ам (единообразие).
4. Парсер STRUCTURE-ALL-GUIDES.md (см. ниже) — пересобрать после правки источников.

### 🟡 01-04-structure-guide-N.md — разметка глубины упоминания

**Проблема.** Строки «Понятия:» дают плоский список без указания, где понятие введено и где использовано.

**Лечение.** Переразметить в формат из 06-concept-graph-architecture.md § 3.С2:

```markdown
**Понятия:**
- **вводится:** Защищённый слот → `U.Method` (PD.METHOD.021)
- **используется:** Культура ученика — см. 1.S1.SS2
- **противопоставлен:** мотивация
- **сослан:** Слот саморазвития → 1.S1.SS4
```

Принципы:
- Каждое понятие = ровно одно `вводится` во всей программе.
- `используется` обязан указывать на подраздел `вводится`.
- Кейсы и аналогии — НЕ в этой строке, а в теле подраздела.

### 🟢 04-concept-map-guide-4.md — трёхуровневый подход к Tame Flow

**Проблема.** Блок «Tame Flow: управление потоками и ограничениями» (~25 концептов) не был дифференцирован: всё шло либо в Pack, либо как cited. Это ложная дилемма.

**Решение (трёхуровневое, вместо «всё или ничего»):**

| Уровень | Что включает | Как оформить |
|---------|-------------|--------------|
| **Ядро персональной работы** | WIP, Flow Time, Bottleneck, Full Kitting, Управление тремя потоками | Создать `PD.METHOD.023-tame-flow-personal` в `PACK-personal/03-methods/` — адаптация Tame Flow для личной работы. Эти понятия получают `source: PD.METHOD.023`. |
| **Cited (внешний источник)** | Detailing, Expedite, Buffer, EC, Critical Chain (Goldratt) — детальные инструменты, которые руководство использует как теорию | В concept-map пометить `source: cited:Tendon/Goldratt`. В Pack-ontology **не идут**. В тексте — как раздел «Теория» со ссылкой на первоисточник. |
| **Кейсы** | Хохланд, история Danaher | В текст подраздела как иллюстрация, **не в «Понятия:»**. |

Хохланд → `source: case` → в текст подраздела.
Решение принимает пилот; от него зависит, нужен ли `PD.METHOD.023` или достаточно только cited + кейсы.

# Заготовка: Руководство 4 «Введение в системное мышление (ВСМ)»

> **Лозунг:** «Научись видеть системы — и переводить проблемы в задачи»
>
> **Объект:** Пилот — инструмент мышления (различения, роли, границы, эмерджентность).
> **Ось:** Мастерство (M4 — мыслительное мастерство как инструмент).
> **Актуально со ступеней:** 3–5.
> **Время на подраздел:** 60 мин чтение + 40 мин практика = **100 мин** (7 подразделов ≈ 11,5 ч/неделю).

---

## Основные идеи (из Pack)

1. **Системное мышление ≠ систематичность** — это инструмент анализа, не порядок.
2. **Различение = основа** — Роль ≠ исполнитель, Система ≠ сумма частей, Метод ≠ инструмент.
3. **Системный подход 1.0 → 2.0 → 3.0** — эволюция: объект → интересы → безмасштабность.
4. **Успешная система** — учитывает интересы всех заинтересованных лиц (не удовлетворяет всех — учитывает).
5. **Эмерджентность** — новые свойства на каждом системном уровне. Предсказуема через моделирование.
6. **Системный разбор** — 4 вопроса: зачем, что, как, когда. Артефакт: системная карта.
7. **IWE как система** — Пилот + Машина = Болид. Пример для практики разбора.
8. **Себя как система** — организм + личность + мастерство. Саморазвитие = изменение системы «я».
9. **Tame Flow: потоки и ограничения** — управление работой как управление системой потоков. Закон Литтла: WIP = Flow Time × Throughput. Flow Efficiency — ключевая метрика системы. Ограничение — не «проблема», а управляемый параметр системы (Five Focusing Steps).

### Из текущих docs (проверка — не забыть)
- **Системное мышление** — мышление с понятиями трансдисциплин интеллект-стека
- **Систематичность** — регулярность, дисциплина (≠ системное мышление)
- **Системность** — способность «разрезать» мир на системы
- **Интеллект-стек** — 16 трансдисциплин, прошивающих интеллект
- **Классический системный подход 1.0** — целостность, эмерджентность, вложенность
- **Системный подход 2.0** — нет интересов → нет системы
- **Системный подход 3.0** — безмасштабность, деантропоморфность, многоуровневая эволюция
- **Система** — объект физического мира, выделенный вниманием
- **Функция** — желаемое действие системы
- **Целевая система / надсистема / подсистема** — иерархия
- **Система создания** — кто создаёт целевую систему
- **Системы в окружении** — ближнее и дальнее окружение
- **Процесс** — взаимодействие систем (≠ система)
- **Проект** — временная деятельность по созданию уникальной системы
- **Мета-системный переход** — переход на более высокий уровень
- **Таблица 3×3** — 3 области интересов ролей × 3 области интересов систем
- **Модель / мета-модель / мульти-модель / мега-модель** — уровни описания
- **Чёрный ящик / прозрачный ящик** — внешнее vs внутреннее описание
- **Интерфейс** — правила взаимодействия системы с миром
- **Характеристика / состояние / показатель** — свойства системы
- **Оргзвено / оргроль / сервис** — организация как система
- **Стадия / этап / итерация / инкремент** — жизненный цикл
- **Tame Flow** — метод операционного менеджмента (Стив Тендон): управление потоками работ через ограничения, метрики и предварительную подготовку

### Tame Flow: управление потоками и ограничениями

**Источник:** Стив Тендон, *Tame Your Work Flow* (2020) / *The Book of TameFlow* (2022). Адаптация для персональной работы — курс «Рациональная работа» (Aisystant), WP-313, D-004.

**Основные понятия:**
- **Закон Литтла (Little's Law)** — WIP = Flow Time × Throughput. Фундаментальное уравнение операционного менеджмента.
- **Flow Time / Lead Time** — время от начала работы до готового результата. Состоит из Touch Time + Wait Time.
- **Touch Time** — время, когда работа реально выполняется (руки/мозг на задаче).
- **Wait Time** — время ожидания: задача готова, но ресурс занят.
- **Flow Efficiency** — Touch Time / Flow Time. Бенчмарки: <15% (без управления), 15–40% (типично), 60%+ (гиперпродуктивность).
- **WIP (Work In Progress)** — незавершённая работа. Ограничение WIP = основной рычаг управления скоростью.
- **Full Kitting** — предварительная подготовка всего необходимого до начала работы (материалы, инструменты, метод, приоритет). Цель: сделать сессию чистым Touch Time.
- **Five Focusing Steps** — 5 шагов теории ограничений (Голдратт): Identify → Exploit → Subordinate → Elevate → Repeat. Алгоритм управления bottleneck.
- **Ограничение (Constraint / Bottleneck)** — фаза/этап/ресурс, определяющий общую пропускную способность системы. Не «проблема», а управляемый параметр.
- **Трихотомия ограничений (Trichotomy)**:
  - **Work Flow constraint** — поперёк всего pipeline (самая длинная загрузка)
  - **Work Process constraint** — внутри этапа (самое длинное среднее потоковое время)
  - **Work Execution constraint** — динамическое, в моменте выполнения
- **Политика vs Ресурс** — дискриминатор (Шрагенхайм): bottleneck стабилен → ресурс; bottleneck мигрирует/elevation не работает → политика.
- **Cognitive freeze** — когнитивное ограничение: рост открытых вопросов без решений, прокрастинация. Mental models = ultimate constraint в knowledge work.
- **Throughput Accounting (T/I/OE)** — Throughput / Inventory / Operating Expense. Незавершённый PR = Inventory, не прогресс.
- **Evaporating Cloud (EC)** — инструмент разрешения конфликтов: 5 элементов (Common Goal, Need A, Need B, Want D, Want D'), скрытые допущения, injection.
- **Negative Branch Reservation (NBR)** — проверка рисков: «3 негативных ветви + обрезка каждой» после любого injection.
- **Три потока личной жизни** (PD.METHOD.023): внешние рабочие продукты, внутренняя информация (обучение + память), внутренняя энергия.

**Связь с системным мышлением:**
- Tame Flow — применение системного подхода к управлению работой: потоки как системы, ограничения как точки воздействия, метрики как показатели состояния.
- В руководстве 4 — как практический инструмент системного разбора собственной деятельности: «Я как система создания» → «Мои потоки» → «Мои ограничения» → «Точки воздействия».

### Риски дублирования
- **Роль** — в руководстве 1 как ролевая траектория, здесь — как системное понятие (функция).
- **Системный подход** — в руководстве 1 как мировоззрение, здесь — как инструмент.
- **Агент** — в руководстве 1 как создатель, здесь — как исполнитель роли.
- **Эмерджентность** — в руководстве 1 как свойство личности, здесь — как системное свойство.

## Чего НЕ должно быть в руководстве 4 (границы)

- Нет методов M001–M008 (это руководство 2)
- Нет инструкций по IWE (это руководство 3)
- Нет мировоззренческой дуги (это руководство 1)
- Нет курсовых элементов

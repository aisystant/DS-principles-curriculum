# Структура: Руководство 1 «Системное саморазвитие»

> **Лозунг:** «Ты - система. Узнай, как она устроена»
> **Время на раздел:** 25 мин чтение + 15 мин практика = 40 мин
> **Всего разделов:** 7
> **Недельный бюджет:** ~4,5 ч (7 × 40 мин)

---

## Раздел 1. Эффект Земмельвейса и культура ученика

```yaml
section_id: PD.GUIDE.1.S1
title: "Эффект Земмельвейса и культура ученика"
order: 1
stage_relevant: [1, 2, 3, 4, 5]
cp_check: [cp.rhy, cp.wld]
bh_check: [bh.awr, bh.stb]
bottleneck_hint: "Если застрял - проверь bh.stb: есть ли защищенный слот саморазвития?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Эффект Земмельвейса | Сопротивление простым, но важным изменениям | Сопротивление изменениям → инерция системы |
| Культура ученика | Базовые практики: защищенный слот, саморегуляция, фиксация | Культура → норма поведения → условие функционирования |
| Допуск к сложности | Базовая культура необходима для применения методов, как асептика для хирурга | Допуск → граница доступа → условие входа |
| Стиль жизни ученика | Саморазвитие как непрерывная норма, не проект | Стиль → паттерн поведения → структура деятельности |
| Гигиенический минимум | 6 столпов восстановления: сон, питание, движение, связи, досуг, смысловая работа | Гигиена → поддержание системы → устойчивость |

### Содержание (план ~2500 слов)

#### Теория / различение
- История Земмельвейса: снижение смертности на 90% за счет простого правила (мыть руки), но врачи сопротивлялись 20 лет
- Различение: культура ≠ мотивация. Мотивация - вспышка, культура - инфраструктура
- Различение: допуск ≠ мастерство. Асептика ≠ хирургия, но без асептики хирургия невозможна

#### Метод / практика
- Защищенный слот: фиксированное время, неизымаемое, как обед или сон
- Саморегуляция: шкала состояния 0–5, действие при спаде
- Фиксация мышления: заметка как минимальный шаг

#### Через IWE
- Автоматический трекинг слотов (WakaTime, календарь)
- Агент-Диагност: алерт при пропуске 2+ слотов подряд
- Pack-норма: ритуал входа в роль через ОРЗ

#### Проверка
- cp.rhy ≥ 2: слот есть и работает 3+ недели
- bh.stb ≥ 2: состояние стабильно (колебания в пределах 2–4)

### Практика (15 мин)
- Аудит текущей недели: сколько слотов саморазвития было? Сколько сорвалось?
- Один защищенный слот на следующую неделю: время, триггер, минимальное действие

## Раздел 2. Созидатель = Болид

```yaml
section_id: PD.GUIDE.1.S2
title: "Созидатель = Болид"
order: 2
stage_relevant: [1, 2, 3, 4, 5]
cp_check: [cp.wld, cp.iwe]
bh_check: [bh.awr, bh.sys]
bottleneck_hint: "Если застрял - проверь cp.iwe: понимает ли пилот Machine как часть себя?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Созидатель | Человек + инструменты + среда, действующий в мире | Субъект деятельности → агент системы |
| Болид | Пилот + Машина, неделимая система | Система → целостность → эмерджентность |
| Пилот | Человек: мышление, мировоззрение, роли | Управляющая подсистема → регулятор |
| Машина | IWE: репозитории, среда, агенты | Исполнительная подсистема → инструмент |
| Мировоззрение | Машинный код: определяет зачем и куда | Программа → цель → направление |
| Методы | Runtime: как действовать | Алгоритм → процесс → способ |
| IWE | Среда исполнения: где происходит работа | Среда → контекст → условие |

### Содержание (план ~3000 слов)

#### Теория / различение
- Болид-онтология: человек не отделен от инструментов (FORM.089 §1)
- Различение: Пилот ≠ Машина, но Болид = Пилот + Машина (не сумма, система)
- Различение: мировоззрение ≠ знания. Знания - данные, мировоззрение - интерпретатор
- Методы работают в среде IWE; без среды методы - теория

#### Метод / практика
- Аудит текущей Machine: что используешь, что нет, что тормозит
- Карта «Пилот ↔ Машина»: что делает человек, что делает среда

#### Через IWE
- Структура IWE в Pack: FORM, METHOD, CAT, SOTA
- Drift-detection: когда Machine расходится с намерениями
- Агент-Навигатор: поддержание траектории

#### Проверка
- cp.wld ≥ 2: пилот понимает себя как систему
- bh.sys ≥ 1: машина работает (есть Pack, есть ОРЗ)

### Практика (15 мин)
- Нарисовать схему своего Болида: что в Пилоте, что в Машине
- Один элемент Машины, который усилит Болида (и шаг подключения)

## Раздел 3. Мировоззренческая дуга

```yaml
section_id: PD.GUIDE.1.S3
title: "Мировоззренческая дуга: 5 фаз"
order: 3
stage_relevant: [1, 2, 3, 4, 5]
cp_check: [cp.wld, cp.int]
bh_check: [bh.awr, bh.agn]
bottleneck_hint: "Если застрял - проверь cp.wld: на какой фазе дуги пилот?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Мировоззренческая дуга | 5 фаз эволюции мировоззрения (FORM.080 §3.1) | Эволюция → развитие → движение |
| Системное мировосприятие | Видеть мир как систему, не сумму фактов | Восприятие → модель → представление |
| Деятельностный кругозор | Широта взглядов, охват доменов | Кругозор → граница знания → охват |
| Агентность | Способность инициировать изменения | Свобода действия → степень влияния |

### Содержание (план ~2500 слов)

#### Теория / различение
- 5 фаз: «Зачем учиться - непонятно» → «Я - система» → «Мир - система, и я в ней деятель»
- Каждая фаза - не ступень, а линза. Пилот может быть на разных фазах в разных доменах
- Переход фаз = кризис осознания, не линейный прогресс

#### Метод / практика
- Диагностика текущей фазы: по 3 признакам для каждой
- Практика «расширения дуги»: чтение вне домена, разговор с «другим миром"

#### Через IWE
- Pack как зеркало фазы: какие разделы читаются легко, какие сопротивляются
- Агент-Диагност: вопросы для определения фазы
- История запросов показывает паттерн мышления

#### Проверка
- cp.wld ≥ 3: пилот осознает свою фазу и может назвать 2 признака
- bh.agn ≥ 2: есть пример инициированного изменения за последний месяц

### Практика (15 мин)
- Определить свою текущую фазу по 3 признакам
- Найти один источник «следующей фазы» и зафиксировать в Pack

## Раздел 4. 5 ролей созидателя

```yaml
section_id: PD.GUIDE.1.S4
title: "5 ролей созидателя"
order: 4
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.skl, cp.cre]
bh_check: [bh.met, bh.inv]
bottleneck_hint: "Если застрял - проверь cp.skl: какая роль развита, какая - нет?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| 5 ролей созидателя | Ученик → Интеллектуал → Профессионал → Исследователь → Просветитель (FORM.087) | Роль → функция → место в системе |
| Ученик | Собирает, пробует, находит методы | Потребитель знания → вход |
| Интеллектуал | Переводит неопределенность в задачи | Преобразователь → процессинг |
| Профессионал | Создает рабочие продукты системно | Производитель → выход |
| Исследователь | Продвигает SoTA, находит новое | Исследователь → расширение границы |
| Просветитель | Передает знание, масштабирует культуру | Передатчик → репликация системы |
| Калибр личности | 6 уровней от Я до человечества (FORM.090) | Масштаб → охват → уровень системы |
| Мастерство | Главный капитал (vs информация) | Компетенция → способность → качество |

### Содержание (план ~3000 слов)

#### Теория / различение
- Роль ≠ должность. Роль = функция, которую выполняешь в данный момент
- Траектория не линейна: пилот может быть Учеником в одном домене и Профессионалом в другом
- Интеллектуал - ключевой перевод: от «интересно» к «надо сделать задачу"
- Мастерство накапливается, информация обесценивается

#### Метод / практика
- Карта ролей по доменам: где какая роль активна
- Практика «ролевого переключения»: ритуал входа в роль (роль, метод, РП, критерии)

#### Через IWE
- Агенты как ролевые усилители: Портной для Интеллектуала, Диагност для Ученика
- Pack: история ролевых переключений, рост по доменам
- RCS-профиль: какая роль доминирует

#### Проверка
- cp.skl ≥ 2: пилот может назвать свою доминирующую роль и 2 домена
- bh.met ≥ 2: есть 2+ метода, применяемых на уровне «умение"

### Практика (15 мин)
- Заполнить карту ролей: 3 домена × 5 ролей (оценка 0–3)
- Выбрать 1 домен и 1 роль для развития в следующем месяце

## Раздел 5. Ступени развития и ворота

```yaml
section_id: PD.GUIDE.1.S5
title: "Ступени развития и ворота"
order: 5
stage_relevant: [1, 2, 3, 4, 5]
cp_check: [cp.rhy, cp.skl, cp.wld]
bh_check: [bh.sys, bh.inv, bh.met]
bottleneck_hint: "Если застрял - проверь bh.sys: есть ли стабильная система за последние 4 недели?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| 5 ступеней | Случайный → Пробующий → Систематический → Проактивный → Мастер (FORM.089) | Ступень → уровень развития → фаза |
| bh-порог | Минимальный уровень поведенческих навыков для перехода | Порог → условие перехода → граница |
| cp-порог | Минимальный уровень cp для перехода | Порог → условие перехода → граница |
| Двойной gate | Аттестатор (bh) + Диагност (cp) - оба должны подтвердить | Контроль → проверка → валидация |
| Bottleneck | Узкое место, которое определяет скорость развития | Ограничение → регулятор потока → bottleneck |
| Окно ступени | Время, за которое пилот проходит ступень (недели) | Временной интервал → период → фаза |

### Содержание (план ~3500 слов)

#### Теория / различение
- Ступени - не грейды, а режимы функционирования
- Каждая ступень имеет фокус: Случайный = хаос/тупик, Пробующий = слот + методы, Систематический = 10ч/нед + мастерство метода, Проактивный = +развитие IWE
- bh.sys + bh.inv - ключевые пороги. Без инвестированного времени переход невозможен
- Окна в неделях: Случайный → Пробующий (2–4), Пробующий → Систематический (8–12), и т.д.

#### Метод / практика
- Диагностика текущей ступени по 4 признакам
- План перехода: какой bh/cp поднять, за какое время
- Bottleneck-анализ: что тормозит прямо сейчас

#### Через IWE
- Автоматический расчет bh/cp по данным WakaTime, Git, Pack
- Аттестатор: отчет по bh за неделю/месяц
- Диагност: вопросы для определения cp

#### Проверка
- bh.sys ≥ порог текущей ступени
- cp.rhy ≥ порог текущей ступени
- bh.inv ≥ минимум для ступени (часы/неделю)

### Практика (15 мин)
- Определить текущую ступень по 4 признакам
- Найти bottleneck: один bh или cp, который блокирует переход
- Зафиксировать в Pack цель на 4 недели

## Раздел 6. Ценности, намерения, творческий конвейер

```yaml
section_id: PD.GUIDE.1.S6
title: "Ценности, намерения и творческий конвейер"
order: 6
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.wld, cp.cre]
bh_check: [bh.awr, bh.agn]
bottleneck_hint: "Если застрял - проверь различение: ценности ≠ цели?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Ценности | Инвариант, машинный код: что важно всегда | Ценность → ось → направление |
| Цели | Производные от ценностей, временные | Цель → точка → состояние |
| Намерения | Направленность деятельности, вытекающая из неудовлетворенностей | Намерение → вектор → движение |
| Неудовлетворенности | Движущая сила: что не так, что хочешь изменить | Разрыв → напряжение → энергия |
| Творческий конвейер | 4 стадии: потребление → размышления → действие → досуг (PD.METHOD.019) | Конвейер → поток → процесс |
| SoTA | Лучшие теории в домене | Знание → состояние системы → точка отсчета |

### Содержание (план ~3000 слов)

#### Теория / различение
- Ценности ≠ цели: ценности - compass, цели - waypoint
- Неудовлетворенности → намерения → стратегия → действия
- Творческий конвейер: дисбаланс = bottleneck. Перекос в потребление = «вечный ученик», перекос в действие = «горе-деятель"
- Досуг - не отдых, а стадия конвейера (восстановление + инкубация)

#### Метод / практика
- Практика «3 неудовлетворенности»: записать, ранжировать, выбрать одну для работы
- Карта конвейера: сколько времени на каждой стадии за неделю

#### Через IWE
- Pack: фиксация ценностей, намерений, неудовлетворенностей
- Агент-Стратег: каскад ВДВ, WP-реестр
- Диагност: алерт при дисбалансе конвейера (>60% на одну стадию)

#### Проверка
- cp.wld ≥ 3: пилот может назвать 3 ценности и отличить от целей
- cp.cre ≥ 2: есть пример прохождения полного конвейера (от потребления до досуга)

### Практика (15 мин)
- Записать 3 неудовлетворенности текущего момента
- Для каждой: какая ценность стоит за ней? Какое намерение возникает?
- Проверить баланс конвейера за прошлую неделю

## Раздел 7. 6 результатов программы и гигиенический минимум

```yaml
section_id: PD.GUIDE.1.S7
title: "6 результатов программы и гигиенический минимум"
order: 7
stage_relevant: [3, 4, 5]
cp_check: [cp.rhy, cp.skl]
bh_check: [bh.stb, bh.met]
bottleneck_hint: "Если застрял - проверь bh.stb: стабильность состояния и режима"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| 6 результатов (FORM.093) | Ясность, агентность, собранность, регулярность, способность производить, продуктивность | Результат → выход системы → состояние |
| Ясность | Понимание себя, своих ролей, направления | Понимание → структура знания → форма |
| Агентность | Способность действовать, не ждать разрешения | Свобода → степень влияния → мощность |
| Собранность | Способность фокусироваться и доводить до конца | Целостность → устойчивость → целое |
| Регулярность | Постоянство практики, не всплески | Ритм → периодичность → структура времени |
| Способность производить | Создавать рабочие продукты, не только потреблять | Производство → трансформация → деятельность |
| Продуктивность | Эффективность: результат / затраченное время | Эффективность → отношение → метрика |
| Гигиенический минимум | 6 столпов: сон, питание, движение, связи, досуг, смысловая работа (PD.METHOD.020) | Инфраструктура → поддержание → база |

### Содержание (план ~3000 слов)

#### Теория / различение
- 6 результатов - не метрики, а наблюдаемые изменения. Проверяются через поведение, не через тесты
- Результат ≠ сертификат. Результат = способность функционировать на новом уровне
- Гигиенический минимум: без него методы работают на 30–50%. Это не «здоровый образ жизни", это инфраструктура созидателя
- Смысловая работа как столп: без неё остальные 5 - поддержание, не развитие

#### Метод / практика
- Аудит 6 результатов: самооценка по 10-балльной шкале
- Аудит гигиенического минимума: 6 столпов, статус каждого
- План восстановления: один столп на месяц

#### Через IWE
- Диагност: еженедельный опрос по 6 результатам
- Pack: журнал состояния, тренды по столпам
- Алерты: пропуск сна 3+ дней, отсутствие досуга 5+ дней

#### Проверка
- cp.rhy ≥ 3: регулярность стабильна (4+ недели)
- bh.stb ≥ 3: состояние в зеленой зоне (3–5) 80% времени
- bh.met ≥ 2: 2+ метода на уровне «умение"

### Практика (15 мин)
- Самооценка по 6 результатам (шкала 0–10)
- Аудит 6 столпов (шкала 0–10)
- Выбрать 1 столп и 1 результат для фокуса на месяц

# Структура: Руководство 2 «Методы саморазвития»

> **Лозунг:** «Найди свой стиль саморазвития»
> **Время на раздел:** 45 мин чтение + 25 мин практика = 70 мин
> **Всего разделов:** 7
> **Недельный бюджет:** ~8 ч (7 × 70 мин)
> **Актуально со ступеней:** 2–5
> **Ось:** Мастерство (M1 - мастерство саморазвития)
> **Объект:** Пилот - методы M001-M009

## Раздел 1. Ввод: метод как форма, культура ученика, bottleneck-фокус

```yaml
section_id: PD.GUIDE.2.S1
title: "Ввод: метод как форма, культура ученика, bottleneck-фокус"
order: 1
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.skl, cp.rhy]
bh_check: [bh.met, bh.inv]
bottleneck_hint: "Если застрял - проверь bh.met: есть ли хотя бы один метод на уровне «умение»?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Метод (M001–M009) | Форма деятельности: как действовать, отделенная от содержания | Форма → структура действия → способ |
| Культура ученика | Базовые практики, необходимые для применения методов: слот, саморегуляция, фиксация (PD.METHOD.021) | Культура → норма поведения → условие функционирования |
| Эффект Земмельвейса | Сопротивление простым, но важным изменениям, даже при наличии доказательств | Сопротивление → инерция системы → торможение |
| Bottleneck-фокус | Концентрация на одном узком месте, которое определяет скорость всей системы | Ограничение → регулятор потока → узкое место |
| Степени мастерства | 4 уровня: Объяснение → Умение → Навык → Мастерство (CAT.003, FORM.006) | Ступень → уровень развития → фаза |
| Форма vs содержание | Метод - это форма (как), знание - это содержание (что) | Форма/содержание → структура/материя → различение |
| IWE как контекст | Экзокортекс как среда применения методов, не замена методу | Среда → контекст → условие |

### Содержание (план ~3000 слов)

#### Теория / различение
- Метод = форма (как я действую), не содержание (что я думаю). Различение из FORM.006: один и тот же метод работает в разных доменах, потому что форма универсальна, а содержание меняется
- Эффект Земмельвейса применительно к методам: простые практики (слот, заметка, pomodoro) дают 90% эффекта, но сопротивление переходу длится месяцы (PD.METHOD.021)
- Различение: метод первичен, элемент культуры работы - производное. Метод работает без IWE (бумага и ручка достаточны), IWE только усиливает
- Bottleneck-фокус: не все 9 методов сразу, а один, поднятие которого даст максимальный эффект для текущей ступени (теория ограничений через призму саморазвития)
- Культура ученика как допуск: без защищенного слота и саморегуляции методы остаются теорией (PD.METHOD.021)

#### Метод / практика
- Степени мастерства (CAT.003): Объяснение (могу рассказать) → Умение (могу сделать под присмотром) → Навык (могу сделать стабильно) → Мастерство (могу адаптировать и учить)
- Диагностика текущей степени по каждому из 9 методов: шкала 0–3
- Как выбрать bottleneck-метод: анализ текущей ступени + bh-индексы + неудовлетворенности
- Минимальный шаг: один метод, один защищенный слот, одна неделя, одна проверка
- Правило «один метод в фокусе»: пока метод не на уровне «умение», не брать следующий

#### Через IWE
- Pack как карта методов: раздел METHOD, статус каждого метода (M001–M009)
- WakaTime: трекинг времени по методам, корреляция bh.inv с прогрессом методов
- Агент-Диагност: ежемесячный аудит методов, алерт при отсутствии прогресса 3+ недели
- Pack-шаблон «Дашборд 9 методов»: визуализация степеней мастерства
- Элемент культуры работы: как IWE-инфраструктура поддерживает методы (роли, агенты, ритуалы)

#### Проверка
- cp.skl ≥ 2: пилот знает 3+ метода из M001–M009 и может объяснить каждый
- bh.met ≥ 1: хотя бы один метод практикуется регулярно (1+ раз в неделю)
- bh.inv ≥ 2: есть защищенный слот для практики методов (3+ часа в неделю)

### Практика (25 мин)
- Аудит текущих методов: что делаю, что нет, на какой степени (шаблон дашборда)
- Определить bottleneck-метод: какой один метод даст максимальный эффект при подъеме на 1 ступень?
- Зафиксировать в Pack: выбор метода, степень мастерства, план на 4 недели

## Раздел 2. M001–M002: Экзокортекс и инвестирование времени

```yaml
section_id: PD.GUIDE.2.S2
title: "M001–M002: Экзокортекс и инвестирование времени"
order: 2
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.iwe, cp.rhy]
bh_check: [bh.inv, bh.stb]
bottleneck_hint: "Если застрял - проверь bh.inv: есть ли защищенный слот и трекинг времени?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Экзокортекс (M001) | Внешняя когнитивная система: память, вычисление, фокус вынесены за пределы тела | Расширение → дополнение субъекта → граница системы |
| Защищенный слот | Фиксированное время, неизъемное для саморазвития, как сон или обед | Слот → граница доступа → защита ресурса |
| Инвестирование времени (M002) | Направленный поток времени с ожидаемой отдачей в мастерство | Инвестиция → ресурс → превращение в капитал |
| 5 классов работ | Потребление, размышления, действие, досуг, восстановление (PD.METHOD.019) | Класс → категория → тип деятельности |
| Pomodoro | Цикл 25 мин фокуса + 5 мин восстановления | Ритм → периодичность → структура времени |
| Наградная помидорка | Pomodoro, завершенный восстановлением как наградой, не просто перерывом | Награда → закрытие цикла → мотивация |
| WakaTime | Автоматический трекинг времени в среде разработки | Инструмент → измерение → метрика |
| Бюджет времени | Распределение недельного времени по 5 классам работ | Бюджет → планирование ресурса → распределение |
| Ритуал входа в роль | Три шага: роль, метод, РП, критерии (FORM.089 §5) | Ритуал → триггер → вход в контекст |

### Содержание (план ~4500 слов)

#### Теория / различение
- Экзокортекс (M001): память вынесена за пределы мозга, мышление распределено между человеком и системой. Различение: экзокортекс ≠ гаджеты. Экзокортекс - это система, гаджеты - только инструменты
- Различение: инвестирование vs трата времени. Трата - поток без отдачи, инвестиция - поток, превращающийся в мастерство (M002)
- 5 классов работ (PD.METHOD.019): потребление (вход), размышления (переработка), действие (выход), досуг (инкубация), восстановление (поддержка). Дисбаланс любого класса = bottleneck
- Различение: защищенный слот ≠ свободное время. Свободное время заполняется случайно, слот - инфраструктура
- Ритуал входа в роль (FORM.089 §5): переход между контекстами требует энергии, ритуал снижает затраты на переключение
- Наградная помидорка: восстановление интегрировано в цикл работы, а не отложено (PD.METHOD.022)

#### Метод / практика
- Защищенный слот: фиксация (время, день, место), неизъемность (правило «слот не переносится, он пропускается»), триггер (сигнал начала)
- Pomodoro: 25/5 для мышления, 50/10 для производства, индивидуальная калибровка длины цикла
- Наградная помидорка: после завершения цикла - осознанное восстановление, не просто отвлечение
- WakaTime: настройка категорий, соответствие 5 классам работ, еженедельный отчет
- Бюджет времени: расчет недельного фонда (168 часов), вычитание неизъемного (сон, работа, транспорт), распределение остатка по 5 классам
- Ритуал входа в роль: 3 минуты перед слотом (роль - метод - РП - критерии готовности)
- Степени мастерства M001: Объяснение (знаю, что такое экзокортекс) → Умение (использую 1+ инструмент) → Навык (система инструментов работает без напряжения) → Мастерство (экзокортекс адаптируется под задачу автоматически)
- Степени мастерства M002: Объяснение (знаю про 5 классов) → Умение (трекаю время 1+ неделю) → Навык (бюджет составляется и соблюдается) → Мастерство (время перераспределяется по bottleneck автоматически)

#### Через IWE
- Pack как ядро экзокортекса (M001): хранилище знаний, методов, форм, каталогов. Различение: Pack ≠ база знаний, Pack = операционная система созидателя
- WakaTime-интеграция: автоматический сбор bh.inv, корреляция с прогрессом методов
- Агент-Диагност: еженедельный отчет по 5 классам работ, алерт при дисбалансе (>60% на один класс)
- Pack-шаблон «Бюджет времени»: план/факт по неделям, отклонения, тренды
- Ритуал ОРЗ как ритуал входа в роль: роль (какую роль принимаю) - метод (какой метод использую) - РП (какой продукт) - критерии (когда готово)
- Автоматический расчет bh.inv по данным WakaTime + Git + Pack

#### Проверка
- cp.iwe ≥ 2: пилот использует экзокортекс системно (Pack, WakaTime, агенты)
- bh.inv ≥ 2: 5+ часов в неделю в защищенных слотах саморазвития
- bh.stb ≥ 2: состояние стабильно во время слотов (колебания 2–4 по шкале 0–5)
- cp.rhy ≥ 2: ритуал входа в роль работает 3+ недели без пропусков

### Практика (25 мин)
- Расчет текущего бюджета времени: недельный фонд, неизъемное, остаток, распределение по 5 классам
- Создание одного защищенного слота: время, день, триггер, минимальное действие, ритуал входа
- Настройка WakaTime: 3 категории, соответствующие 5 классам работ, проверка отчета за текущую неделю

## Раздел 3. M003–M004: Медленное чтение и мышление письмом

```yaml
section_id: PD.GUIDE.2.S3
title: "M003–M004: Медленное чтение и мышление письмом"
order: 3
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.skl, cp.cre]
bh_check: [bh.met, bh.inv]
bottleneck_hint: "Если застрял - проверь bh.met: чтение переводится в заметки или остается потреблением?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Систематическое медленное чтение (M003) | Чтение для мышления: пауза, заметка, связь, а не скорость и объем | Потребление → переработка → вход в конвейер |
| Образовательная заметка | Структурированная фиксация мысли из чтения: источник, тезис, связь, вопрос | Заметка → элемент знания → строительный блок |
| Мышление письмом (M004) | Письмо как процесс мышления, не фиксация результата мышления | Письмо → мышление → процесс познания |
| Черновик | Неформальная запись для себя: мысли, ассоциации, полуформулировки | Черновик → промежуточная форма → разработка |
| Заготовка | Структурированный текст, готовый к доработке в продукт | Заготовка → полуфабрикат → переход к выходу |
| Пост / публикация | Готовый рабочий продукт, предназначенный для аудитории | Публикация → выход → замыкание конвейера |
| Творческий конвейер | 4 стадии: потребление → размышления → действие → досуг (PD.METHOD.019) | Конвейер → поток → процесс создания |
| WP Gate / Формулировка работы | Перевод заметки в рабочий продукт через четкую формулировку (FORM.089 §5) | Ворота → граница перехода → точка решения |

### Содержание (план ~4500 слов)

#### Теория / различение
- Различение M003: чтение для мышления ≠ чтение для информации. Информационное чтение накапливает данные, мыслительное чтение перерабатывает их в структуру
- Различение M004: письмо = мышление, не фиксация результата мышления. Когда пишешь, мысль формируется, а не записывается (FORM.006)
- Творческий конвейер (PD.METHOD.019): M003 = стадия потребления, M004 = стадия размышлений. Без перевода между стадиями конвейер не работает
- Различение: заметка ≠ черновик ≠ заготовка ≠ пост. Каждая форма имеет свою функцию и свои критерии готовности
- Формулировка работы (WP Gate): момент, когда заметка получает имя, критерии готовности и срок (FORM.089 §5)
- Образовательная заметка как минимальная единица входа в конвейер: без неё чтение остается потреблением

#### Метод / практика
- Медленное чтение: правило паузы (остановиться после каждого значимого тезиса), правило связи (что это перекликается с тем, что я уже знаю), правило вопроса (что осталось непонятным)
- Образовательная заметка: структура из 4 полей - Источник (что читал), Тезис (главная мысль), Связь (с чем соотносится), Вопрос (что проверить/найти)
- Мышление письмом: заметка → черновик → заготовка → пост. Каждый переход - WP Gate с критериями готовности
- Черновик: пишется для себя, критерий готовности - «я понял, что думаю»
- Заготовка: пишется для аудитории, критерий готовности - «структура ясна, осталась полировка»
- Пост: критерий готовности - «можно опубликовать»
- Минимальный шаг M003: одна образовательная заметка за одно чтение (даже если прочитано 3 страницы)
- Минимальный шаг M004: 15 минут письма без редактирования (freewriting) по заметке
- Степени мастерства M003: Объяснение (знаю, что такое медленное чтение) → Умение (делаю заметку из прочитанного) → Навык (систематически связываю с SoTA) → Мастерство (читаю с позиции «что я могу добавить в SoTA»)
- Степени мастерства M004: Объяснение (знаю, что такое мышление письмом) → Умение (пишу черновик по заметке) → Навык (регулярно перевожу черновики в заготовки) → Мастерство (письмо становится основным инструментом мышления)

#### Через IWE
- Pack как хранилище образовательных заметок: связь с CAT (каталоги), FORM (формы), METHOD (методы)
- Агент-Портной: проверка образовательной заметки на структуру (4 поля), поиск связей с существующими заметками
- WP Gate в Pack: шаблон формулировки работы, автоматическая проверка критериев готовности
- История заметок: трек прогресса по M003-M004, количество заметок → черновиков → заготовок → постов
- Агент-Диагност: алерт при накоплении 10+ заметок без перевода в черновики (bottleneck конвейера)
- Интеграция с SoTA: образовательная заметка автоматически связывается с разделом SoTA в Pack

#### Проверка
- cp.skl ≥ 2: пилот делает образовательные заметки системно (2+ в неделю)
- bh.met ≥ 2: медленное чтение практикуется 1+ раз в неделю
- cp.cre ≥ 2: есть пример перевода заметки в черновик или заготовку за последний месяц
- bh.inv ≥ 2: время на чтение и письмо защищено в бюджете

### Практика (25 мин)
- Прочитать 3 страницы с паузами: остановиться после каждого значимого тезиса
- Оформить одну образовательную заметку по шаблону (Источник, Тезис, Связь, Вопрос)
- Проверить в Pack: есть ли заметка, с которой можно установить связь? Если нет - зафиксировать вопрос для поиска

## Раздел 4. M005–M006: Мышление проговариванием и досуг

```yaml
section_id: PD.GUIDE.2.S4
title: "M005–M006: Мышление проговариванием и досуг"
order: 4
stage_relevant: [2, 3, 4, 5]
cp_check: [cp.skl, cp.wld]
bh_check: [bh.met, bh.stb]
bottleneck_hint: "Если застрял - проверь bh.stb: есть ли осознанное восстановление в расписании?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Мышление проговариванием (M005) | Использование речи как инструмента мышления, не только коммуникации | Речь → мышление → процесс познания |
| Проверка на бытовой язык | Тест понимания: можешь ли объяснить сложное простыми словами | Проверка → валидация → критерий готовности |
| Голосовая заметка | Аудиозапись мысли вслух с последующей транскрибацией | Заметка → фиксация → вход в систему |
| Организация досуга (M006) | Осознанное планирование восстановления, не заполнение пауз | Досуг → восстановление → поддержка системы |
| Осознанное восстановление | Досуг по выбору и намерению, не по инерции или избеганию | Восстановление → управление состоянием → регуляция |
| Эшелонированный досуг | Три уровня: микро (5–15 мин), мезо (вечер), макро (отпуск) | Эшелон → уровень → масштаб |
| Работа с состоянием (PD.METHOD.022) | Шкала состояния 0–5, распознавание спада, действие при падении | Состояние → регулятор → обратная связь |
| Творческий конвейер: досуг | Досуг как 4-я стадия конвейера: инкубация и восстановление (PD.METHOD.019) | Досуг → стадия процесса → замыкание цикла |

### Содержание (план ~3500 слов)

#### Теория / различение
- Мышление проговариванием (M005): речь активирует другие когнитивные механизмы, чем письмо. Различение: проговаривание ≠ обсуждение. Обсуждение - коммуникация, проговаривание - мышление вслух
- Проверка на бытовой язык: если не можешь объяснить «бабушке» - не понял (критерий Фейнмана как метод M005)
- Различение M006: досуг ≠ отдых. Отдых - пассивное восстановление, досуг - активное, осознанное, по выбору. Досуг = стадия конвейера (PD.METHOD.019), не перерыв между стадиями
- Осознанное восстановление: досуг планируется и выбирается, а не происходит по умолчанию (соцсети, телевидение - не досуг, если не выбраны намеренно)
- Эшелонированный досуг: микро (между помидорками), мезо (вечернее восстановление), макро (выходные, отпуск). Без микро-досуга нет устойчивости, без макро - выгорание
- Работа с состоянием (PD.METHOD.022): состояние определяет качество мышления, не наоборот. Шкала 0–5: 0-1 (остановка), 2 (только рутина), 3 (стандартная работа), 4 (сложные задачи), 5 (пиковая производительность)
- Наградная помидорка как микро-досуг: восстановление интегрировано в рабочий цикл, не отложено

#### Метод / практика
- Голосовые заметки: мысль вслух → запись → транскрибация → текстовая заметка. Минимальный шаг: 3 минуты на тему «что я сейчас изучаю»
- Практика «объяснение ребенку»: выбрать концепцию из Pack, записать 2-минутное объяснение простыми словами
- Проверка на бытовой язык: если в объяснении есть жаргон - заменить; если нет примера - придумать; если нет связи с жизнью - найти
- Эшелонированный досуг: планирование на 3 уровня (микро - каждый день, мезо - каждую неделю, макро - каждый квартал)
- Работа с состоянием: ежедневная оценка по шкале 0–5, действие при спаде ниже 3 (сокращение слота, смена метода, микро-досуг)
- Минимальный шаг M005: одна голосовая заметка в неделю с транскрибацией
- Минимальный шаг M006: один мезо-досуг в неделю, запланированный заранее
- Степени мастерства M005: Объяснение (знаю, что такое мышление проговариванием) → Умение (записываю голосовые заметки) → Навык (регулярно проверяю на бытовой язык) → Мастерство (проговаривание - равноправный канал мышления с письмом)
- Степени мастерства M006: Объяснение (знаю про эшелоны досуга) → Умение (планирую мезо-досуг) → Навык (баланс микро/мезо/макро поддерживается) → Мастерство (досуг как стадия конвейера работает автоматически)

#### Через IWE
- Голосовые заметки → Pack: автотранскрибация, связь с образовательными заметками, поиск по тексту
- Агент-Диагност: ежедневный опрос состояния (шкала 0–5), алерт при тренде вниз 3+ дней
- Pack: журнал досуга (микро/мезо/макро), проверка баланса, корреляция с bh.stb
- Трекинг восстановления через WakaTime: категория «восстановление», процент от общего времени
- Агент-Диагност: алерт при отсутствии мезо-досуга 5+ дней или макро-досуга 30+ дней
- Ритуал закрытия дня как мезо-досуг: 10 минут на восстановление, фиксация в Pack

#### Проверка
- cp.skl ≥ 2: пилот использует голосовые заметки или проговаривание 1+ раз в неделю
- bh.stb ≥ 2: состояние стабильно (колебания 2–4) 80% времени
- bh.met ≥ 2: досуг планируется осознанно (не по инерции)
- cp.wld ≥ 2: пилот различает досуг и отдых, понимает роль восстановления в конвейере

### Практика (25 мин)
- Записать 3-минутную голосовую заметку на тему «что я сейчас изучаю»
- Проверить на бытовой язык: можно ли объяснить это неподготовленному человеку? Найти 1 жаргонное слово и заменить
- Запланировать один мезо-досуг на неделю: время, активность, намерение (для чего)
- Оценить свое состояние сейчас по шкале 0–5 и выбрать действие

## Раздел 5. M007–M008: Окружение и стратегирование

```yaml
section_id: PD.GUIDE.2.S5
title: "M007–M008: Окружение и стратегирование"
order: 5
stage_relevant: [3, 4, 5]
cp_check: [cp.wld, cp.cre]
bh_check: [bh.agn, bh.sys]
bottleneck_hint: "Если застрял - проверь cp.wld: ясны ли неудовлетворенности и намерения?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Формирование окружения (M007) | Сознательное управление системой влияний: кто усиливает, кто тормозит, кто отражает | Окружение → система влияний → контекст |
| Агент-усилитель | Человек, среда или практика, которые увеличивают эффект методов | Усиление → положительная обратная связь → рост |
| Агент-тормоз | Человек, среда или практика, которые снижают эффект методов | Торможение → отрицательная обратная связь → спад |
| Агент-зеркало | Человек или среда, которые отражают текущее состояние пилота | Зеркало → рефлексия → самопознание |
| Стратегирование (M008) | Переход от неудовлетворенностей к намерениям и стратегии | Стратегия → направление → вектор развития |
| Неудовлетворенности | Движущая сила: разрыв между «есть» и «надо» | Разрыв → напряжение → энергия изменения |
| Намерения | Направленность деятельности, вытекающая из неудовлетворенностей | Намерение → вектор → движение |
| Каскад ВДВ | Вектор → Домен → Веха: иерархия стратегических элементов | Каскад → уровни детализации → структура плана |
| Управление тремя потоками | Поток работ, поток времени, поток ценности (PD.METHOD.023, Tame Flow) | Поток → движение ресурса → процесс |

### Содержание (план ~4000 слов)

#### Теория / различение
- Формирование окружения (M007): окружение ≠ список людей. Окружение - это паттерн взаимодействий, который усиливает или ослабляет методы. Различение: агент окружения ≠ друг. Агент - функция, друг - роль
- Три типа агентов окружения: усилители (увеличивают эффект методов), тормоза (блокируют или размывают), зеркала (отражают состояние и дают обратную связь)
- Стратегирование (M008): различение стратегия ≠ план. Стратегия = направление (куда), план = маршрут (как и когда). Различение: намерение ≠ цель. Намерение = вектор, цель = точка
- Неудовлетворенности → намерения → стратегия: логика движения от разрыва к действию (FORM.089 §3)
- Каскад ВДВ: Вектор (главное направление) → Домен (область применения) → Веха (контрольная точка). Каждый уровень детализирует предыдущий
- Управление тремя потоками (PD.METHOD.023): поток работ (что делаю), поток времени (когда), поток ценности (зачем). Bottleneck в любом потоке останавливает всю систему (Tame Flow)
- Различение: SoTA (лучшие теории в домене) ≠ стратегия. SoTA - точка отсчета, стратегия - направление движения от нее

#### Метод / практика
- Аудит окружения: составить список 10+ контактов/сред, классифицировать по типу (усилитель/тормоз/зеркало/нейтральный)
- Практика «один разговор в неделю» с агентом-усилителем: фиксация темы, инсайтов, действий
- Работа с тормозами: не «удалить из жизни», а «минимизировать влияние» или «переквалифицировать»
- Техника «неудовлетворенность → намерение»: формулировка за 5 минут (что не так → что хочу → какой вектор)
- Каскад ВДВ: минимальный шаг - один вектор (1 предложение), один домен (1 область), одна веха (1 результат за квартал)
- Управление тремя потоками: еженедельная проверка - где bottleneck? В работе (слишком много начато), во времени (нет слотов), в ценности (непонятно зачем)
- Степени мастерства M007: Объяснение (знаю про типы агентов) → Умение (провел аудит окружения) → Навык (регулярно управляю влиянием тормозов) → Мастерство (окружение системно настроено на усиление методов)
- Степени мастерства M008: Объяснение (знаю про каскад ВДВ) → Умение (сформулировал один вектор) → Навык (каскад работает на 2+ уровнях) → Мастерство (стратегирование - регулярная практика, не разовое упражнение)

#### Через IWE
- Pack: карта окружения (агенты, типы, частота контакта, влияние), обновление раз в квартал
- Агент-Стратег: каскад ВДВ, WP-реестр, проверка связности векторов с рабочими продуктами
- Агент-Диагност: опрос по неудовлетворенностям, ранжирование, связь с намерениями
- Трекинг трех потоков: Pack (поток ценности), WakaTime (поток времени), WP-реестр (поток работ)
- Агент-Диагност: алерт при несвязанности WP с вектором ВДВ (поток работ и поток ценности расходятся)
- Ритуал стратегической сессии: еженедельный/ежемесячный разбор каскада ВДВ через Pack

#### Проверка
- cp.wld ≥ 3: пилот может назвать 2 неудовлетворенности и соответствующие намерения/векторы
- bh.agn ≥ 2: есть пример инициированного изменения за последний месяц (разговор с агентом, корректировка окружения)
- bh.sys ≥ 2: окружение систематизировано (карта есть, действия запланированы), каскад ВДВ зафиксирован в Pack
- cp.cre ≥ 2: есть рабочий продукт, связанный с вектором ВДВ

### Практика (25 мин)
- Нарисовать карту окружения: минимум 3 усилителя, 2 тормоза, 1 зеркало. Отметить, кто из тормозов может быть переквалифицирован
- Сформулировать одну неудовлетворенность → намерение → вектор ВДВ (1 предложение на каждый уровень)
- Проверить в Pack: зафиксирован ли каскад? Если нет - создать запись

## Раздел 6. M009: Планирование и каскад

```yaml
section_id: PD.GUIDE.2.S6
title: "M009: Планирование и каскад"
order: 6
stage_relevant: [3, 4, 5]
cp_check: [cp.rhy, cp.cre]

bottleneck_hint: "Если застрял - проверь bh.sys: работает ли каскад на уровне недели и дня?"
```

### Ключевые понятия (граф → meta-meta)

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Планирование (M009) | Метод превращения намерений в расписанные действия | План → ориентир → структура времени |
| Каскад планирования | Иерархия: год → квартал → месяц → неделя → день | Каскад → уровни детализации → структура |
| Год / квартал / месяц | Уровни стратегического планирования: направление, приоритеты, ресурсы | Уровень → масштаб → охват |
| Неделя / день | Уровни оперативного планирования: действия, слоты, триггеры | Уровень → детализация → исполнение |
| WP Gate / Формулировка работы | Перевод намерения в рабочий продукт через четкую формулировку (FORM.089 §5) | Ворота → точка решения → начало работы |
| Ритуал входа в роль | Три шага перед слотом: роль, метод, РП, критерии (FORM.089 §5) | Ритуал → триггер → вход в контекст |
| Двойной gate | Аттестатор (bh) + Диагност (cp) подтверждают готовность к переходу | Контроль → проверка → валидация |
| Отклонение / корректировка | Различение план/факт, анализ отклонений, адаптация | Обратная связь → корректировка → устойчивость |

### Содержание (план ~3500 слов)

#### Теория / различение
- Планирование как метод (M009), не инструмент. Различение: план ≠ прогноз. Прогноз предсказывает будущее, план создает ориентиры для действия в неопределенности (FORM.006)
- Различение: планирование ≠ стратегирование. Стратегирование (M008) отвечает на «зачем и куда», планирование (M009) - на «что и когда». Без стратегии план - список дел, без плана стратегия - мечта
- Каскад: каждый уровень детализирует предыдущий. Год задает направление и лимиты, квартал - фокусы, месяц - проекты, неделя - действия, день - исполнение
- Различение: цель планирования ≠ выполнение плана. Цель планирования - снижение неопределенности и фокусировка. 70% выполнения хорошего плана лучше 100% выполнения плохого
- WP Gate: момент, когда план переходит в действие. Формулировка работы (роль, метод, РП, критерии, срок) - условие входа в работу (FORM.089 §5)
- Двойной gate на стыке планирования и действия: аттестатор проверяет bh (есть ли ресурс), диагност проверяет cp (понятно ли что и зачем)
- Отклонение как норма: план всегда отличается от факта, важна корректировка, не точность прогноза

#### Метод / практика
- Каскад год → квартал → месяц → неделя → день: каждый уровень имеет свой формат, свою детализацию, свои критерии готовности
- Год: 3 вектора ВДВ, 3 фокуса, лимиты (время, деньги, внимание)
- Квартал: 1 фокус из 3 годовых, 2-3 проекта, метрики
- Месяц: проекты → WP, распределение по неделям, бюджет времени
- Неделя: обзор конвейера, 3 приоритета, баланс 5 классов работ, защищенные слоты
- День: 3 приоритета, ритуал входа в роль, буфер (20% времени на неожиданности)
- Ритуал входа в роль: перед каждым слотом - 3 минуты (роль, метод, РП, критерии). Повторение ритуала = снижение затрат на переключение
- План недели: проводится в конце предыдущей недели или начале текущей. Обзор: что сделано, что перенесено, что нового
- План дня: проводится утром или вечером. Правило «3 приоритета»: не больше 3 значимых дел в день
- Степени мастерства M009: Объяснение (знаю про каскад) → Умение (планирую неделю и день) → Навык (каскад работает на 4+ уровнях, отклонения анализируются) → Мастерство (планирование адаптируется под bottleneck автоматически)

#### Через IWE
- Pack: шаблоны каскада (год, квартал, месяц, неделя, день), связь между уровнями (годовой вектор → месячный проект → недельный WP)
- Агент-Стратег: автокаскад, проверка связности (все ли WP связаны с векторами ВДВ?), алерт при висячих WP
- Агент-Диагност: отчет по выполнению плана (план/факт), анализ отклонений, тренды
- WP-реестр: все рабочие продукты с формулировкой, статусом, сроком, связью с каскадом
- Интеграция с календарем: защищенные слоты отображаются в календаре, ритуал входа в роль - как напоминание
- Ритуал ОРЗ как планирование дня: роль (какую принимаю) - метод (какой использую) - РП (какой продукт) - критерии (когда готово)

#### Проверка
- cp.rhy ≥ 3: планирование стабильно (4+ недели без пропусков плана недели и дня)
- bh.inv ≥ 3: время инвестируется согласно плану (отклонение <20% от бюджета)
- bh.sys ≥ 3: каскад работает на 2+ уровнях (неделя + день минимум, желательно + месяц)
- cp.cre ≥ 2: есть рабочие продукты, созданные по плану (не спонтанно)

### Практика (25 мин)
- Заполнить шаблон «Неделя»: 3 приоритета, защищенные слоты, баланс 5 классов работ
- Проверить связь с месячным планом (если есть) или с вектором ВДВ (если месячного нет)
- Настроить ритуал входа в роль для одного слота на неделе: написать формулировку (роль, метод, РП, критерии)
- Запланировать день на завтра: 3 приоритета, буфер 20%, ритуал входа для каждого значимого слота

| Понятие | Определение | Meta-meta связь (FPF) |
|---------|-------------|----------------------|
| Личный стиль саморазвития | Индивидуальная комбинация методов, их приоритетов и степеней мастерства | Стиль → индивидуальная форма → паттерн |
| Система методов | 9 методов как взаимосвязанная структура, не чек-лист | Система → целостность → эмерджентность |
| Bottleneck-метод | Один метод, подъем которого на 1 ступень даст максимальный эффект для системы | Ограничение → регулятор потока → узкое место |
| Through IWE | Реализация метода через экзокортекс: агенты, инструменты, шаблоны Pack | Контекст → условие → усиление |
| Мастерство саморазвития (M1) | Способность выбирать, применять и адаптировать методы под себя | Мастерство → мета-навык → регуляция развития |
| Элемент культуры работы | Производное от метода: привычка, норма, ритуал, интегрированная в IWE | Культура → стабильная форма → инфраструктура |
| Дрейф (drift) | Постепенное размывание метода без регулярной проверки | Дрейф → отклонение → деградация |
| Ротация фокуса | Смена bottleneck-метода по мере развития системы | Ротация → переключение → динамика |

### Содержание (план ~4000 слов)

#### Теория / различение
- 9 методов как система, не чек-лист. Различение: набор ≠ система. Набор - перечень элементов, система - взаимосвязи между ними. M002 (время) без M009 (планирование) - трата, M003 (чтение) без M004 (письмо) - потребление (PD.METHOD.019)
- Личный стиль: каждый пилот имеет свою комбинацию. Один мыслит письмом (M004), другой - проговариванием (M005). Один восстанавливается физически (M006), другой - интеллектуально. Стиль не копируется, а открывается через практику
- Bottleneck-метод: не «слабейший метод», а тот, подъем которого разблокирует остальные. Если нет слотов (M002) - не работает ничего. Если нет стратегии (M008) - план (M009) случаен. Если нет экзокортекса (M001) - методы не накапливаются
- Through IWE: каждый метод усиливается экзокортексом, но метод первичен. Различение: усиление ≠ замена. IWE не делает метод за пилота, IWE снижает трение и увеличивает скорость
- Элемент культуры работы: когда метод отработан до «навыка», он становится элементом культуры. Культура - это методы, работающие без напоминаний (PD.METHOD.021)
- Дрейф методов: без регулярной проверки методы размываются. Защищенный слот сдвигается, медленное чтение превращается в скоростное, досуг - в инерцию. Проверка раз в месяц - минимум для стабильности
- Ротация фокуса: bottleneck меняется по мере развития. На ступени 2 bottleneck - M002 (время). На ступени 3 - M004 (мышление письмом) или M009 (планирование). На ступени 4 - M007 (окружение) или M008 (стратегирование)

#### Метод / практика
- Аудит 9 методов: дашборд со статусом каждого (не начат / объяснение / умение / навык / мастерство). Цветовая кодировка bottleneck (красный - блокирует, желтый - тормозит, зеленый - работает)
- Выбор bottleneck-метода на квартал: 1 метод в фокусе, 2-3 в поддержке, остальные - на поддержании текущего уровня
- Построение личного стиля: 3 метода во фокусе (главные каналы мышления и действия), 3 в поддержке (работают на автомате), 3 в ожидании (не начаты или на уровне объяснения)
- Проверка раз в месяц: что изменилось, что дрейфнуло, какой метод стал новым bottleneck'ом
- Through IWE для каждого метода: какой агент, какой инструмент, какой шаблон Pack усиливает именно этот метод
- Минимальная система для ступени 2: M002 (слот) + M003/M004 (вход в конвейер) + M009 (план дня/недели)
- Минимальная система для ступени 3: + M001 (экзокортекс системен) + M005 (второй канал мышления) + M006 (восстановление)
- Минимальная система для ступени 4: + M007 (окружение) + M008 (стратегирование) + интеграция всех 9

#### Через IWE
- Pack: дашборд 9 методов, тренды по степеням мастерства, история bottleneck'ов
- Агент-Диагност: ежемесячный аудит методов, рассчет bottleneck-метода по bh/cp
- Агент-Портной: рекомендации по bottleneck-методу с учетом ступени и стиля пилота
- WakaTime + Pack: корреляция времени (bh.inv) и прогресса методов (bh.met)
- Автоматический отчет «Методы»: изменения за месяц, дрейф, рекомендации по фокусу
- Through IWE - таблица для каждого метода:
  - M001: Pack (ядро), WakaTime (трекинг), агенты (память)
  - M002: WakaTime (классы работ), Pack (бюджет), календарь (слоты)
  - M003: Pack (образовательные заметки), агент (проверка связей)
  - M004: Pack (черновики, заготовки), WP Gate (формулировка)
  - M005: Голосовые заметки → Pack (транскрибация)
  - M006: Pack (журнал досуга), агент (опрос состояния)
  - M007: Pack (карта окружения), агент (аудит контактов)
  - M008: Pack (каскад ВДВ), агент-Стратег (векторы)
  - M009: Pack (шаблоны каскада), WP-реестр, агент (план/факт)

#### Проверка
- cp.skl ≥ 3: пилот знает все 9 методов и может объяснить 5+
- bh.met ≥ 3: 3+ метода на уровне «умение» или выше
- cp.iwe ≥ 3: IWE используется для усиления методов, не замены
- bh.inv ≥ 3: время распределено по методам согласно плану
- bh.sys ≥ 3: система методов стабильна (3+ месяца без критического дрейфа)
- cp.rhy ≥ 3: регулярная проверка методов (ежемесячный аудит)

### Практика (25 мин)
- Заполнить дашборд 9 методов: статус по каждому (не начат / объяснение / умение / навык / мастерство)
- Определить bottleneck-метод: какой один метод сейчас блокирует или тормозит остальные?
- Выбрать 1 bottleneck-метод на следующий месяц и зафиксировать в Pack
- Для выбранного метода определить Through IWE: какой агент, какой инструмент, какой шаблон Pack помогут именно сейчас?
- Проверить дрейф: какой метод за последний месяц опустился на ступень ниже? Что с этим делать?

### В. Логичность распределения

**Bug 1: алфавитная сортировка разделов.** Во всех 4 руководствах разделы идут S1, S10, S2, S3, … вместо S1, S2, …, S10. Артефакт сортировки строк. Чинить в STRUCTURE-ALL-GUIDES.md (численная сортировка) и проверить генератор.

**Bug 2: пропуски подразделов.** Контракт `00-requirements.md` — 7 подразделов на раздел. Фактически:

| Руководство | Раздел | Пропущенные SS |
|-------------|--------|----------------|
| Guide 2 | S1, S3, S4 | SS05 во всех трёх |
| Guide 3 | S2, S3 | SS05 |
| Guide 3 | S4 | SS01 |
| Guide 4 | S2 | SS01, SS03 |
| Guide 4 | S3 | SS06 |
| Guide 4 | S8, S9 | SS07 |

Итого ≥11 пустых слотов из 280. Решение: либо заполнить, либо явно пометить «факультативный слот / расширение Портного».

**Bug 3: концепт-листы Guide 4 повреждены.** Многие подразделы Guide 4 содержат строки вида:

```
- Понятия: дисциплина; порядок в действиях)
- Понятия: фиксирующая границу между понятиями); не данная природой); прошивающих интеллект)
```

Имена понятий усечены, остались хвосты пояснений в скобках и непарные `)`. Похоже на ошибку парсинга при сборке. **Срочное действие:** проверить исходник `04-structure-guide-4.md` и парсер.

**Дублирование без разметки глубины:** одни и те же понятия в нескольких подразделах без указания, где введены и где используются:

### Д. Связь с мета-мета-понятиями

**Мета-мета-понятия = понятия трансдисциплин** из [SPF.SPEC.002 § 2](../../../DS-Knowledge-Index-Tseren/docs/SPF/ontology.md) (U.System, U.Episteme, U.Method, U.RoleAssignment, U.Characteristic, U.Flow, ...). Каждое доменное понятие ЛР обязано иметь родителя из этого списка (правило SPF.SPEC.002 § 5 п.2). Доменный Граф понятий ЛР — `PACK-personal/ontology.md` § 2.

**Резолюция родителей (U.*) для понятий из v4-руководств:**

| Понятие | Родитель (U.* из SPF) | Комментарий |
|---------|------------------------|-------------|
| Творческий конвейер | `U.System` | Холон с границей и потоками работ (потребление → размышления → действие → досуг) |
| Защищённый слот | `U.Method` | Способность регулярно создавать инфраструктуру внимания |
| Каскад ВДВ | `U.Method` | Процедура планирования: вектор → домен → веха |
| Active Inference | `U.Episteme` | Научная теория как единица знания |
| Эпистемический граф | `U.Episteme` | Артефакт знания со структурой |
| Гигиенический минимум | `U.Method` | Шесть столпов восстановления как воспроизводимая практика |
| Knowledge Extraction | `U.Method` | Процедура извлечения паттернов |
| Калибр личности | `U.Characteristic` | Измеряемая ось оценки созидателя (соответствует ст. 5 ролей) |
| Состояние | `U.Flow` | Временный режим функционирования (уже в `PACK-personal/ontology.md`) |
| Роль в деятельности (Ученик, Интеллектуал, …) | `U.RoleAssignment` | Контекстная функция в деятельности |
| Мировоззрение | `U.Episteme` | Метафреймворк интерпретации (уже в Pack) |
| Учёт времени | `U.Method` | Метод регистрации времени (уже в Pack) |
| Pack | `U.BoundedContext` | Семантическая рамка со словарём и инвариантами |
| Граница системы | `U.Boundary` | Разделение внутреннего и внешнего |
| Ступень развития | `U.Kind` | Классификационная единица |
| Day Close / Сессия | `U.Work` | Запись свершившегося действия |
| WeekPlan / DayPlan | `U.WorkPlan` | Расписание намерений |
| Ценность как принцип | `U.Commitment` | Деонтический объект |

**Важно (FPF A.7):** в строку «Понятия:» подраздела попадают только сущности из `PACK-personal/ontology.md` § 2. Кейсы и аналогии — НЕ понятия.

| Что | Правильное место | Примеры из STRUCTURE-ALL-GUIDES |
|-----|------------------|----------------------------------|
| **Кейс** (история, случай) | В тексте подраздела как иллюстрация | Эффект Земмельвейса, Хохланд, Tame Flow |
| **Аналогия** (метафора-перенос) | В тексте подраздела как метафора | Болид (аналогия Созидателя), асептика для хирурга, рабочий стол в хаосе |

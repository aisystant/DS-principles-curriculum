# Формальные требования к универсальным руководствам v4

## Общие параметры

| Параметр | Значение |
|----------|----------|
| **Формат** | Справочник (reference) + структура для Портного. Домашки, контрольные вопросы и задания — в персональном руководстве (адаптируются Портным), но опираются на универсальные шаблоны из руководств |
| **Длина** | 2000–5000 слов на **подраздел**. Руководство целиком — 14 000–35 000 слов (7 подразделов) |
| **Тон** | Инструктивный, не дидактический. «Вот как устроено» — не «вот что ты должен». Запрет на ИИ-стиль: нет воды, нет «важно отметить», нет «стоит рассмотреть». Короткие предложения, конкретика. |
| **Типографика** | Запрет на длинное тире (—). Использовать дефис (-) или двоеточие. Пример: «не ассистент — среда» → «не ассистент, а среда» или «не ассистент: среда». |
| **Самодостаточность** | Каждый подраздел читается без контекста других подразделов. Кросс-ссылки — в конце, не в середине текста. Повторы минимальны: если понятие уже определено — краткая напоминание + ссылка, не полное определение |
| **Связь с Pack** | Каждое утверждение ссылается на FORM.NNN или METHOD.NNN |
| **Связь с Рабочим развитием** | В конце каждого руководства — раздел «Что дальше»: как это руководство связано с программой «Рабочее развитие» и какие методы/навыки переходят туда |
| **Проверка на FPF** | Перед финализацией — аудит онтологии: все понятия проверяются на дублирование с FPF, SPF, ZP. Чистота терминологии: одно слово = одно понятие, нет омонимов без явного различения |

## Онтологическая целостность

Для каждого руководства формируется список основных понятий (15–30 шт.). Каждое понятие:

1. **Проверяется на наличие в графе понятий Pack** (PD.FORM.022 и др.).
2. **Если отсутствует** — вносится в граф с определением и связями.
3. **Проверяется связь с мета-мета-понятием** (FPF-уровень): понятие не должно висеть в воздухе без связи с фундаментальной онтологией.
4. **Проверяется на дублирование:** одно слово = одно понятие, нет омонимов без явного различения (HD).

## Структура раздела (контракт)

Каждое руководство состоит из **разделов** (~5–10 шт.). Каждый раздел:

- Рассчитан на **1 неделю** изучения в персональном руководстве
- Содержит **7 подразделов** (1 на каждый день недели)
- Каждый подраздел рассчитан на фиксированное время изучения **включая практику** (чтение + задание):

| Руководство | Чтение | Практика | Полный подраздел | Недельный бюджет |
|-------------|--------|----------|------------------|------------------|
| 1 «Системное саморазвитие» | 25 мин | 15 мин | **40 мин** | ~4,5 ч |
| 2 «Методы саморазвития» | 45 мин | 25 мин | **70 мин** | ~8 ч |
| 3 «IWE: работа и развитие» | 45 мин | 30 мин | **75 мин** | ~8,5 ч |
| 4 «Введение в системное мышление (ВСМ)» | 60 мин | 40 мин | **100 мин** | ~11,5 ч |

> **Адаптация Портным:** указанное время — ориентир для универсального руководства. Портной удваивает или сокращает по cp.rhy и bh.stb пилота. Практика всегда есть: чтение без действия = развлечение.

## Структура подраздела (шаблон)

```
### Подраздел N.N. Название

**Время:** X часов
**Что узнаешь:** 1–2 предложения

#### Теория / различение
- 2–3 ключевых различения из Pack (ссылки FORM.NNN)
- Каждое: определение + пример + типичная ошибка

#### Метод / практика
- Что это, зачем, минимальный шаг
- Степени мастерства (CAT.003): объясняю → умею → привычка → передаю

#### Через IWE
- Как материал реализуется через IWE
- Конкретные инструменты, команды, шаблоны
- Ссылки на PD.FORM.095 (Event Model)

#### Проверка
- cp-индекс + bh-порог для этого подраздела
- Bottleneck-подсказка: «если застрял — проверь cp.X»

#### Ссылки
- Кросс-ссылки на другие руководства
- Ссылки на FORM, METHOD, CAT
```

## Формальные требования к стилю

1. **Домашки, контрольные вопросы и задания** — в универсальном руководстве в виде **шаблонов** (что проверить, какое задание типичное). Адаптация под конкретного пилота — в персональном руководстве (Портной). Нельзя оставлять руководство без практики: чтение без действия = развлечение.
2. **Справочный порядок:** читать в любом порядке, искать по нужде. Но есть рекомендуемый порядок от Портного.
2. **Справочный порядок:** читать в любом порядке, искать по нужде.
3. **Метка «Актуально со ступени N»** — frontmatter раздела, не отдельный раздел внутри руководства.
4. **Приложение «Через IWE»** — в каждом руководстве, кроме руководства 3 (IWE само по себе про IWE).
5. **cp/bh-проверки** — обязательны в каждом подразделе. Без них нет способа измерить прогресс.
6. **Bottleneck-подсказка** — в конце каждого раздела: «если застрял здесь — проверь cp.X».

## Чеклист приёмки руководства

- [ ] Каждое утверждение имеет ссылку на FORM.NNN или METHOD.NNN
- [ ] Все cp/bh-проверки корректны (соответствуют FORM.089 v4)
- [ ] Bottleneck-подсказка есть в каждом разделе
- [ ] Приложение «Через IWE» есть (кроме руководства 3)
- [ ] Кросс-ссылки на другие руководства работают
- [ ] Читается без контекста других руководств
- [ ] Нет дрейфа от Pack PD (verified против FORM.NNN)

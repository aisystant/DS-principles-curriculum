# Формальные требования к универсальным руководствам v4

## Общие параметры

| Параметр | Значение |
|----------|----------|
| **Формат** | Справочник (reference) + структура для Портного. Домашки, контрольные вопросы и задания — в персональном руководстве (адаптируются Портным), но опираются на универсальные шаблоны из руководств |
| **Длина** | 2000–5000 слов на **подраздел**. Руководство целиком — 14 000–35 000 слов (7 подразделов) |
| **Тон** | Инструктивный, не дидактический. «Вот как устроено» — не «вот что ты должен». Запрет на ИИ-стиль: нет воды, нет «важно отметить», нет «стоит рассмотреть». Делиться, а не учить: личный опыт вместо советов, вопрос вместо императива, уязвимость вместо экспертной позиции. |
| **Типографика** | Длинное тире (—) допустимо только в двух случаях: (1) пояснение через «это», (2) противопоставление. Не более 1 тире на 3 абзаца. В остальных случаях — дефис (-), двоеточие, скобки или запятая. |
| **Названия подразделов** | Без шифров (bh, cp), без спецтерминологии (gate, bottleneck), без английских слов (S1/S2, runtime). Человеческие названия, понятные широкой аудитории. Плохо: «bh/cp-пороги и двойной gate». Хорошо: «Ворота развития: когда переходить дальше». |
| **Синтаксис** | Предложения длинные, 15-40 слов. Преобладают сложноподчиненные конструкции с причинно-следственными связями. Цепочки через «поэтому», «при этом», «то есть», «а также». Вводные слова: «Вероятно», «Конечно», «При этом», «В итоге». Допустимы грамматические шероховатости, характерные для живой мысли. |
| **Абзацы** | Крупные: 4-8 предложений. Не дробить текст на однострочные абзацы. Списки только при необходимости перечисления. Текст идет монолитными абзацами, структурирован мыслью, а не визуальными разделителями. |
| **Образовательная лексика** | Не использовать: школа, вуз, студенты, преподаватель, обучение, учеба. Замены: участник/пользователь, наставник, развитие/практика. |
| **Самодостаточность** | Каждый подраздел читается без контекста других подразделов. Кросс-ссылки — в конце, не в середине текста. Повторы минимальны: если понятие уже определено — краткая напоминание + ссылка, не полное определение |
| **Связь с Pack** | Каждое утверждение ссылается на FORM.NNN или METHOD.NNN |
| **Связь с Рабочим развитием** | В конце каждого руководства — раздел «Что дальше»: как это руководство связано с программой «Рабочее развитие» и какие методы/навыки переходят туда |
| **Проверка на FPF** | Перед финализацией — аудит онтологии: все понятия проверяются на дублирование с FPF, SPF, ZP. Чистота терминологии: одно слово = одно понятие, нет омонимов без явного различения |

## Онтологическая целостность

Для каждого руководства формируется список основных понятий (15–30 шт.). Каждое понятие:

1. **Проверяется на наличие в графе понятий Pack** (PD.FORM.022 и др.).
2. **Если отсутствует** — вносится в граф с определением и связями.
3. **Проверяется связь с мета-мета-понятием** (FPF-уровень): понятие не должно висеть в воздухе без связи с фундаментальной онтологией.
4. **Проверяется на дублирование:** одно слово = одно понятие, нет омонимов без явного различения (HD).

## Структура раздела (контракт)

Каждое руководство состоит из **разделов** (~5–10 шт.). Каждый раздел:

- Рассчитан на **1 неделю** изучения в персональном руководстве
- Содержит **7 подразделов** (1 на каждый день недели)
- Каждый подраздел рассчитан на фиксированное время изучения **включая практику** (чтение + задание):

| Руководство | Чтение | Практика | Полный подраздел | Недельный бюджет |
|-------------|--------|----------|------------------|------------------|
| 1 «Системное саморазвитие» | 25 мин | 15 мин | **40 мин** | ~4,5 ч |
| 2 «Методы саморазвития» | 45 мин | 25 мин | **70 мин** | ~8 ч |
| 3 «IWE: работа и развитие» | 45 мин | 30 мин | **75 мин** | ~8,5 ч |
| 4 «Введение в системное мышление (ВСМ)» | 60 мин | 40 мин | **100 мин** | ~11,5 ч |

> **Адаптация Портным:** указанное время — ориентир для универсального руководства. Портной удваивает или сокращает по cp.rhy и bh.stb пилота. Практика всегда есть: чтение без действия = развлечение.

## Структура подраздела (шаблон)

```
### Подраздел N.N. Название

**Время:** X часов
**Что узнаешь:** 1–2 предложения

#### Теория / различение
- 2–3 ключевых различения из Pack (ссылки FORM.NNN)
- Каждое: определение + пример + типичная ошибка

#### Метод / практика
- Что это, зачем, минимальный шаг
- Степени мастерства (CAT.003): объясняю → умею → привычка → передаю

#### Через IWE
- Как материал реализуется через IWE
- Конкретные инструменты, команды, шаблоны
- Ссылки на PD.FORM.095 (Event Model)

#### Проверка
- cp-индекс + bh-порог для этого подраздела
- Bottleneck-подсказка: «если застрял — проверь cp.X»

#### Ссылки
- Кросс-ссылки на другие руководства
- Ссылки на FORM, METHOD, CAT
```

## Стилистический эталон письма (адаптирован из CLAUDE.md Индекса знаний)

### Голос и позиция

Автор руководства — **мыслитель-практик, который думает вслух**. Не отстраненный эксперт и не коуч. Инструктивный тон: «Вот как устроено» — не «вот что ты должен». Личный опыт вместо советов: «я заметил» вместо «записывайте». Вопрос вместо императива. Уязвимость вместо экспертной позиции.

### Зачин и финал подраздела

- **Зачин:** первое предложение подраздела содержит ключевую мысль. Без раскачки, без «в этом разделе вы узнаете». Три паттерна: утверждение-определение, личный факт, обобщение через наблюдение.
- **Финал:** обобщающий абзац с расширением масштаба (от личного к общему). Часто конструкция «Таким образом...» с переформулировкой основного тезиса.

### Структура текста

- Подзаголовки внутри подраздела оформляются **жирным текстом** внутри абзаца, а не markdown-заголовками H2/H3. H2/H3 — только для разделов и подразделов.
- Абзацы крупные: 4-8 предложений. Не дробить текст на однострочные абзацы.
- Списки только когда действительно есть перечисление (методы, этапы, критерии).

### Синтаксис

- Предложения длинные, 15-40 слов. Преобладают сложноподчиненные конструкции с причинно-следственными связями.
- Цепочечные конструкции через «поэтому», «при этом», «то есть», «а также».
- Частые вводные слова: «Вероятно», «Конечно», «При этом», «В итоге», «То есть».
- Скобки для уточнений и примеров. Двоеточия перед раскрытием тезиса.
- Длинное тире (—) допустимо только в двух случаях: (1) пояснение через «это», (2) противопоставление. Не более 1 тире на 3 абзаца. В остальных случаях — дефис, двоеточие, скобки или запятая.
- Допустимы грамматические шероховатости, характерные для живой мысли.

### Лексика

- **Доменный словарь** (использовать свободно): мастерство, жизненное мастерство, агентность, деятель, системные уровни, мировоззрение, экзокортекс, неудовлетворенности, калибр личности, мышление письмом, рабочий продукт, роль ученика, конвейер, практика, метод.
- **Книжно-разговорный микс.** Основа книжная, публицистическая. Разговорные вкрапления создают живость: «собрать себя в кучку», «не заморачиваться», «белка в колесе».
- **«Я» используется свободно** в примерах и вводных историях. В основном тексте может переходить на «мы» (инклюзивное) или «вы».
- **Запрет образовательной лексики:** не использовать «школа», «вуз», «студенты», «преподаватель», «обучение», «учеба». Замены: участник/пользователь, наставник, развитие/практика.

### Приёмы (обязательные)

1. **Развернутые аналогии** из бытовой жизни: автомобиль, плавание, горные лыжи, квартира. Минимум одна аналогия на подраздел.
2. **Примеры из личного опыта** или типичного опыта участников. Конкретные, с деталями.
3. **Статистика как аргумент:** конкретные цифры, данные, шкалы.
4. **Ссылки на авторитетов** (конкретные, с именами): Левенчук, Дойч, Талеб, Канеман, Чиксентмихайи.
5. **Противопоставления** для различений: деятель vs потребитель, осознанное vs неосознанное, мастерство vs кругозор.
6. **Риторические вопросы** (умеренно, 1-2 на подраздел).
7. **Цитаты участников** как доказательство: «один из участников говорит так: ...».
8. **Снятие пафоса юмором:** «Иначе как же жить без мечты? Шутка.»

### Форматирование

- **Жирный** для ключевых тезисов и терминов внутри абзаца: «**Мастерство дает скорость обучения**». Якоря для скользящего взгляда.
- **Курсив** почти не используется.
- **Ссылки** обильные, на FORM/METHOD/CAT. Создают внутреннюю сеть понятий.
- **Блок-цитаты** (>) для выводов-резюме и цитат авторитетов.

## Формальные требования к стилю

1. **Домашки, контрольные вопросы и задания** — в универсальном руководстве в виде **шаблонов** (что проверить, какое задание типичное). Адаптация под конкретного пилота — в персональном руководстве (Портной). Нельзя оставлять руководство без практики: чтение без действия = развлечение.
2. **Справочный порядок:** читать в любом порядке, искать по нужде. Но есть рекомендуемый порядок от Портного.
3. **Метка «Актуально со ступени N»** — frontmatter раздела, не отдельный раздел внутри руководства.
4. **Приложение «Через IWE»** — в руководствах 3 и 4 (где участники уже знакомы с IWE). В руководствах 1 и 2 — не использовать: задания не привязаны к IWE.
5. **Проверка понимания** — в каждом подразделе: 2-3 вопроса без шифров cp/bh, на человеческом языке.
6. **Bottleneck-подсказка** — в конце каждого раздела: «если застрял здесь — проверь X». Без шифров.

## Чеклист приёмки руководства

- [ ] Каждое утверждение имеет ссылку на FORM.NNN или METHOD.NNN
- [ ] Все cp/bh-проверки корректны (соответствуют FORM.089 v4)
- [ ] Bottleneck-подсказка есть в каждом разделе
- [ ] Приложение «Через IWE» есть (кроме руководства 3)
- [ ] Кросс-ссылки на другие руководства работают
- [ ] Читается без контекста других руководств
- [ ] Нет дрейфа от Pack PD (verified против FORM.NNN)

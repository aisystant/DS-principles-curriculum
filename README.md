# DS-principles-curriculum

> **Тип репозитория:** Downstream/surface
> **Upstream:** PACK-education (EDU) + ZP (нулевые принципы) + FPF (первые принципы)

Конкретные программы обучения **принципам мышления** (ZP и FPF). Применение универсальных методов обучения из Pack EDU к конкретному предмету — нулевым и первым принципам.

## Как это работает

```
Pack ZP         Pack EDU                 DS (этот репо)
(ЧТО учить)    (КАК учить)              (конкретные программы)
┌───────────┐  ┌──────────────────────┐  ┌──────────────────────────────┐
│ ZP.1-ZP.6 │  │ Шаблон ячейки        │  │ cells/ZP.1-axiomaticity.md   │
│ FPF.*     │─▶│ Методы (M.001-008)   │─▶│ cells/ZP.2-structure.md      │
│           │  │ TPM (профиль)        │  │ tests/ZP.1-questions.md      │
│           │  │ Диагност (R.006)     │  │ programs/iwe-17-22.md        │
│           │  │ Конструктор (FORM.007)│  │ programs/children-7-10.md    │
└───────────┘  └──────────────────────┘  └──────────────────────────────┘
  Принципы        Абстрактные методы       Заполненные ячейки + программы
```

**Правило:** Pack EDU = source-of-truth для методов. ZP = source-of-truth для принципов. Этот репо = их пересечение (конкретное применение).

## Что здесь есть

### Ячейки (cells/)

Каждый файл = один принцип на всех глубинах Bloom (1-5). Каждая ячейка заполняется по **стандарту** (10 полей):

| # | Поле | Что описывает |
|---|------|--------------|
| 1 | `principle` / `depth` / `bloom_level` | Координаты ячейки в матрице Принцип × Bloom |
| 2 | `can_do` | Что ученик **может делать** на этой глубине (наблюдаемые действия) |
| 3 | `methods.primary` | Основные методы обучения (ссылки на EDU.M.*) |
| 4 | `methods.meta` | Мета-методы (Retrieval + Spaced — всегда) |
| 5 | `prerequisites` | Пререквизиты: по тому же принципу, кросс-принципные, когнитивный уровень |
| 6 | `assessment` | Тип оценки, критерии, тест на перенос, тест на удержание |
| 7 | `common_errors` | Типичные ошибки и заблуждения на этой глубине |
| 8 | `domains` | Домены применения (от быта до профессиональных) |
| 9 | `forms` | Конкретные формы занятий по когнитивным уровням (preoperational → postformal) |
| 10 | `bridge_template` | Мостик-фраза: связь с предыдущей глубиной |

### Тесты (tests/)

Банк вопросов для адаптивного теста принципов (методология — EDU.WP.005). Каждый вопрос привязан к can-do из ячейки.

### Программы (programs/)

Готовые программы для конкретных аудиторий:

| Программа | Аудитория | Стратегия | Статус |
|-----------|-----------|-----------|--------|
| kindergarten-3-6 | Дошкольники (детский сад) | Breadth-first, глубина 1, preoperational forms | Планируется |
| children-7-10 | Младшие школьники | Breadth-first, глубина 1-2 | Планируется |
| teens-14-17 | Старшие школьники | Breadth→Bottleneck, глубина 3-4 | Планируется |
| iwe-17-22 | Молодые взрослые | Bottleneck-first, глубина 3-5 | Планируется |
| parent-child | Пары (родитель + ребёнок) | Параллельный | Планируется |

#### Две стратегии обучения

**До ~10 лет (детский сад + младшие школьники)** — **универсальный маршрут.** Все принципы на глубине 1-2, breadth-first. Причина: нет достаточной когнитивной базы для глубины 3+, и нет смысла искать «узкое горлышко» — все принципы одинаково новые. Ключевое: привычка задавать вопросы, различать базовые категории, играть с принципами.

**С ~11 лет и взрослые** — **персональный маршрут через узкое горлышко.** У каждого ученика разный профиль: один принцип может быть на глубине 4, другой — на 1. Эффективное обучение = диагностика профиля → поиск bottleneck-принципа → целевая работа на следующей глубине.

> Для персонального маршрута нужна **роль Диагноста** (см. ниже).

#### Роль: Диагност принципов

Отдельная роль (для преподавателя, ИИ-агента или самого ученика на высоких глубинах) — **распознавание затыков и построение персонального маршрута:**

1. **Профилирование** — определить текущую глубину ученика по каждому принципу (через тесты из `tests/`)
2. **Поиск bottleneck** — найти принцип, который блокирует продвижение по другим (через граф `prerequisites.cross_principle`)
3. **Маршрутизация** — выбрать следующую ячейку для работы (bottleneck-first, а не breadth-first)
4. **Мониторинг** — отслеживать прогресс, корректировать маршрут при смене bottleneck

> Навык диагностики тренируется постепенно: сначала преподаватель/ИИ диагностирует за ученика, затем ученик учится самодиагностике (= глубина 4-5 по принципам). Методология — EDU.R.006 (Диагност).

## Capture-to-Pack: путь нового мема в CAT.001

> **Контекст:** Пользователь T4 (Проактивный) может предложить новый непродуктивный мем. Этот путь описывает, как предложение превращается в карточку каталога.

### Кто может предложить

Пользователь **ступени T4 (Проактивный)** — уже прошедший программу, видит паттерны в своём окружении, чьи мемы уже скомпилированы (глубина 3) в большинстве областей.

### Шаг 1. Фиксация предложения

Пользователь сообщает боту: «Хочу предложить мем». Бот запрашивает:
- Формулировка мема (короткая фраза от первого лица, как убеждение)
- Область (1–5)
- Почему это непродуктивный мем — в чём вред для развития?
- Антитезис — какое убеждение его заменяет?

Бот сохраняет предложение в `inbox платформы` (ЦД L2, таблица `meme_proposals`): статус `submitted`.

### Шаг 2. Первичный фильтр (автоматически)

Система проверяет:
- Дубликат? (семантическое сравнение с существующими мемами)
- Достаточно ли специфичен? (не слишком широкий, не слишком узкий)
- Область определена?

Если дубликат → пользователь получает уведомление: «Похожий мем уже есть: M-XXX». Статус → `duplicate`.
Если не прошёл фильтр → статус `needs_clarification`, бот запрашивает уточнение.

### Шаг 3. Ревью куратора

Куратор (еженедельно, в рамках ревью каталогов) просматривает `meme_proposals` со статусом `submitted` или `clarified`.

**Критерии принятия:**
- Мем реально встречается у нескольких людей (не единичный кейс)
- Имеет чёткий антитезис (заменяемое убеждение)
- Вписывается в логику программы (одна из 5 областей)
- Не дублирует существующие мемы по смыслу

**Решение куратора:**
- `approved` → переходит к шагу 4
- `rejected` → статус `rejected` + причина (пользователь получает уведомление)

### Шаг 4. Создание карточки

Куратор создаёт карточку по шаблону:

```markdown
---
id: CAT.001.M-{NNN}
name: {формулировка мема}
area: {1-5}
source: community_proposal | {имя предложившего, если T4 дал согласие}
status: current
---

# CAT.001 — M-{NNN}: {формулировка}

**Диагностика:** {как распознать этот мем у себя — 2-3 признака}

**Антитезис:** {продуктивное убеждение, которое его заменяет}

---

## Глубина 1 — Осознание
...

## Глубина 2 — Различение
...

## Глубина 3 — Компиляция
...
```

Новый ID присваивается по порядку (следующий после M-064 → M-065).

### Шаг 5. Публикация

1. Карточка добавляется в `DS-principles-curriculum/data/curriculum/CAT.001/`
2. README обновляется (строка в таблице области)
3. `meme_proposals` → статус `published`, ссылка на карточку
4. Пользователю-автору → уведомление: «Твой мем M-065 добавлен в каталог»
5. Commit: `feat(cat001): add M-065 — {краткое название мема}`

### Правила

| # | Правило |
|---|---------|
| R-C2P.1 | Только T4 может предлагать мемы. Другие ступени — через `lesson_rating` |
| R-C2P.2 | Куратор — финальный арбитр. Автоматический фильтр — только предотбор |
| R-C2P.3 | Карточка публикуется только с полными 3 глубинами. Неполная → `draft`, не попадает в Портного |
| R-C2P.4 | Источник фиксируется в frontmatter. Имя автора — только с явного согласия |
